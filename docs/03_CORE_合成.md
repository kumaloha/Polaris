# 03 CORE：合成（动物两维合成）

> 本文档定义合成 Core 玩法。**Core 是可插拔小游戏,通过资源流接入 Meta（见 02）。**
> **这是备选的第二 Core,后置**（主线是消除,见 `04`）——待消除主线验证成功后,用它验证 Meta 框架与算法引擎的复用性（见 `00` §6 / `05` §五）。
> 设计依据对照 `01_设计哲学.md`,Meta 对接见 `02_META系统.md`,技术接口见 `05_技术架构.md`。

---

## 一、核心一句话

动物主题的两维合成：**相同动物合成升级（low floor,谁都会）,但「物种 + 进化阶段」两维参与,凑出高级组合需要规划（high ceiling,有最优解）。**

这是相对现有合成游戏的微创新点：现有合成几乎都是「单维匹配」（相同数字/相同物品就合）；本作是「两维匹配」（物种 + 进化阶段都参与），策略空间更大。

---

## 二、合成单位：动物的两维结构

每个动物单位有两个维度：

- **物种（类别维度）**：猫、狗、兔、狐……（建议 4-6 种核心物种,像扑克 4 花色那样少而清晰）。对应「构筑/搭配路线」。
- **进化阶段（等级维度）**：幼体 → 成体 → 进化体 → ……（一条升级链）。对应「合成升级」。

**视觉要求（守 low floor）**：
- 物种：4-6 种,辨识清晰,别太多（太多升高门槛）。
- 进化阶段：视觉上一眼分高低（体型递增 / 明显进化特征）,不能让玩家猜谁更高级。

---

## 三、合成规则（要定的三件事）

合成规则 = 回答：①什么能合 ②怎么操作 ③合出什么。以下给出方案与待定项。

### 3.1 什么能合（匹配条件）——本作的微创新核心

**基础合成（管 low floor）**：
- 相同物种 + 相同进化阶段的两个 → 合成该物种的下一进化阶段。
- 例：两只「幼体猫」→ 一只「成体猫」。（像 2048,但带物种）
- 这条简单直觉,保证大众秒懂、随便合也能玩。

**进阶合成（管 high ceiling,两维深度的来源）**：
> 这是本作策略深度的关键,需在原型中重点验证。给出几个候选方向,选其一或组合：
- **候选 A：跨物种高级合成**——不同物种的成体,按特定组合可合成「稀有动物」（高价值）。逼玩家既要升等级又要凑物种组合。
- **候选 B：同物种链 + 物种协同**——同物种升到顶可进化成「该物种的王」；场上同时存在多物种成体时触发协同加成。
- **候选 C：物种相性**——某些物种之间合成产出特殊效果（类似花色同花的概念）。

**两维取舍（策略来源）**：让「升等级」和「凑物种组合」不能同时轻松达成 → 逼规划。凑出「高进化 + 特定物种组合」的稀有动物 = 两维都满足 = 最难、最高价值（high ceiling）。

### 3.2 怎么操作（操作方式）——待定,建议拖拽或滑动

| 方式 | 说明 | 适配度 |
|---|---|---|
| 拖拽合成 | 手动把一个拖到另一个上合成（Merge 主流,自由度高）| 推荐,适合策略规划 |
| 滑动合成 | 整盘往一方向滑,相同的撞合（2048 式,每滑全盘变化）| 推荐,策略性强 |
| 掉落合成 | 从顶部掉落/放置,相邻相同合（Suika 式,带物理）| 不太推荐,偏物理手感,与策略最优化方向不搭 |

> 建议在拖拽 / 滑动中二选一,原型可各做一版对比手感。

### 3.3 合出什么（产物逻辑）——建议带分支

- **单线升级**：同物种沿进化链升级（基础,low floor）。
- **分支/转化**：进阶合成产出稀有动物 / 触发特殊效果（深度,high ceiling）。
- 建议：基础单线（简单）+ 进阶分支（深度）并存,形成 LFHC 梯度。

---

## 四、Low Floor / High Ceiling 在合成里怎么体现

| | 体现 |
|---|---|
| **Low Floor** | 相同动物合升级,谁都会;乱合也能不断升级、能玩下去;无角色/铭文也能正常玩 |
| **High Ceiling** | 凑「高进化 + 特定物种组合」的稀有动物需规划;每个局面有「最优合成路径」（算法可算）;同题竞技比谁合得更优 |

**关键**：基础合成必须低到「不动脑也能玩」（守地板）;高级组合是「想钻研的人追求的上限」（高天花板）。两者并存,不互相挡路。

## 五、Core 与 Meta 的对接（资源流）

**Core 产出 → Meta**：
- 分数（合成等级越高、组合越优,分越高）→ 用于解锁抽卡机会、上排行榜。
- 资源 → 用于铭文树。

**Meta 注入 → Core**：
- 当前携带的「角色能力」（一个）在本局生效（如预知下 N 个动物、时间倒流、借贷等）。
- 「铭文加成」（一组数值）在本局生效（如更大场地、更多撤销、更多产出）。

**注意**：Core 不实现角色能力和铭文的逻辑归属,只提供「能力作用的接口」。能力/铭文的定义和管理在 Meta。具体接口见 05。

## 六、算法在合成 Core 的应用（护城河）

- **关卡 = 一段「动物出现序列」**。算法对每个序列：
  - 验证有解（玩家一定能正常合、不死局）→ 保 low floor。
  - 算出最优解（最高分/最优组合路径）→ 定义 high ceiling。
  - 量化难度（可行解与最优解的差距、分支复杂度）→ 个性化推送、分级。
- **个性化**：给累的人轻松序列,给高手有挑战的序列。
- 动物物种有限（4-6 种）+ 进化阶段有限 → 序列空间可控,最优解可算,难度可量化（有限性是算法的朋友）。

## 七、原型要验证的核心问题

做最小可玩版本,优先回答：

1. **基础合成（相同动物升级）光秃秃玩,好不好玩?想不想合下一个?**（验证 low floor 和核心爽点）
2. **两维进阶合成（凑物种组合）有没有「想规划」的乐趣?还是觉得烦?**（验证 high ceiling 是否成立、是否吓跑大众）
3. **一个角色能力（如预知）加进来,是不是「锦上添花」而非「没它玩不了」?**（验证生死线）
4. **「玩 core 赚分 → 解锁抽卡」的循环跑不跑得通、有没有动力?**（验证核心循环）

> 别等系统全做完再验。先用最糙的方式（甚至纸面/极简 demo）把「基础合成 + 一两个进阶组合 + 一个角色能力 + 分数→抽卡」跑起来,让真人玩,看反应。

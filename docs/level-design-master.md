# 关卡设计总纲（Master）

> 2026-06-15 · 这是关卡重做的**权威总纲**，整合并扩展了 `docs/level-design-formula.md`（后者保留了我们一路的分歧/纠错记录，仍可查阅）。
> 本文回答四件事：**我们的目标、关卡设计的本质、关卡设计的方法、整体设计**——并把关卡放进一个**六维全景**里，让设计有依据、全面。

---

## 0. 目标再定位

- **产品**：gem 消除 match-3（交换相邻、3+ 连消、带重力下落 + 顶部补给）。
- **品类**：**hybrid casual**，不是纯消消乐。纯三消天花板低、机会被巨头封死。
- **规模**：奔**上千关**，是持续运营(live ops)，不是一锤子买卖。
- **两条留存线（缺一不可）**：① **关卡设计本身带留存**(本文主攻)；② **萌宠养成带留存**(后续立项，非 RPG 战力养成)。
- **差异化**：头部竞品是"工业长跑 + 稀释换皮"；我们要"**精酿短篇的密度 + 工业长跑的规模**"——靠生成系统在规模上维持设计感，**拒绝换皮**。
- **核心矛盾(=项目全部价值)**：**规模 × 设计感天然对立**。解开它，就是这个项目的意义。
- **不被带偏**：旧 levels.json(糊弄生成)、旧 overlay 概念、旧 C++"宝石排布"思路——全部抛弃。

---

## 1. 六维全景：关卡不是孤立客体（本次新增·让设计全面有依据）

我们之前只深耕"客体"一维。一个关卡活在六个维度里，每一维都得纳入：

| 维度 | 管什么 | 我们的立场/缺口 |
|---|---|---|
| **① 客体**(关卡本身) | 玩法/难度地理/题眼/障碍/目标/特殊宝石 | 深耕区；`settings` 长在这（见 §4） |
| **② 主体**(玩家) | 玩家类型、技能成长、**情绪弧**、认知负荷 | 难度是"相对玩家"的锚；情绪弧=留存来源 |
| **③ 时间** | 单关时长(1-3min)、会话节奏(生命)、留存时间结构、内容生命周期 | match-3 碎片化是硬约束 |
| **④ 感官**(美学) | 主题母题、配色、视觉层次、**动画成本**、juice 爽感 | 四位一体的"美/主题"两位；最便宜的新鲜感 |
| **⑤ 经济**(变现) | 道具/生命、卡关变现、"差一点"、**公平 vs 逼氪** | hybrid=留存×变现，难度墙=变现曲线 |
| **⑥ 数据**(运营) | 通关率/流失/转化、玩家模拟器、A/B、**数据再平衡** | "模拟运营"里"运营"那一半 |

**原则**：每设计一关，这六维都要被回答一遍——不是只把客体做好。玩家完形地体验所有维度。

## 2. 关卡设计的本质

### 2.1 每关是"一道题"，不是"一个数字"
独特局面，要玩家*理解+应对*，不是模板换数字。
**判据**：有一个"无脑乱消"绕不过的题眼；抽掉它难度就塌。

### 2.2 gem 三元素 = 一条"控制力"光谱（随机 → 控制）
- **普通宝石**=可互换的纯颜色载体 → 随机性之源。
- **特殊宝石**=玩家用技巧从随机里炼出的可控能量 → 随机→控制的桥。
- **特殊技能**=棋盘外的确定性力量 → 确定性之源。
- **好玩 = 在随机里用技巧创造控制、把混沌驯化成秩序**。两极(纯随机=老虎机/纯确定=解谜)都不好玩。

### 2.3 难度地理 / 补给水文（地形+障碍+棋子分布）
gem 有**重力**，决定玩法的不是初始帧，是**消除后补给怎么流动**：
- **地形=河道**(宽口=上游补给足；窄口=咽喉；下游=死水)。
- **障碍=堰坝/礁石**(改变补给 + 本身常是目标)；障碍的难 = 硬度 × 它在水文图上的位置。
- **棋子分布=初始水位/下手点**。
- **设计一关 = 画一张水文图，把目标钉在你要它难的水文位置**。死水区=控制力最难到达处。

### 2.4 一关的骨架是"四位一体的形状"（最高本质）
玩家**完形地**一瞬间形成印象，割裂优化难度=格局小。一关骨架是个**形状**，**设计感=这形状同时是四样东西**：
**难度地理(玩法) × 视觉构图(美) × 直觉指引(可读) × 主题表达(风味)**。
四维共振于同一形状，而非各自达标。（沙漏关四样全干：下游死水/对称美/窄腰喊"下面难"/可挂时之沙主题。）

### 2.5 情绪是本质的一部分（补·主体维度）
玩家在一关里走一条**情绪弧**：开局上手→中段紧张/卡顿→破局顿悟→收尾爽快清场。**好关卡是设计这条弧，不只是设计难度**。全程烧脑无爽点=失败品（match-3 本质要"爽"）。

## 3. 关卡设计的方法

### 3.1 设计感三杠杆
**局面**(静态障碍塑地形) + **活变量**(动态障碍/活元素加压力) + **咬合**(特殊宝石组合涌现)。高级关卡=三者叠加。

### 3.2 双难度 + 相对化系数（反数字膨胀）
- **认知难度**(要想多久看懂怎么破) vs **执行难度**(看懂了手上做不做得到)。平庸只抠步数(执行)，高级在认知做文章。**设计感≈认知难度**。
- 参数全部相对化：步数=理论最小步数 × **S**；分数=每步密度 **D** × 步数；障碍=覆盖率 **O** + 种类 **K**(≤3)。

### 3.3 题眼矩阵 = 目标类型 × 阻碍方式
- **目标类型**(赢什么)：清除 / 收集 / 运输(需出口) /（凑分少用）。
- **阻碍方式**(怎么难)：远征 / 围城 / 隔断 / 收割 / 攻坚 / 揭示 / 精算 / 钥匙 / 连环…
- 二维交叉=题眼空间；新鲜感主要靠"换目标类型"。

### 3.4 外部竞品研究抽象原则（只作方法尺子）
1. **复杂度阶梯**：难度=复杂度逐级累加，不是拔数值。
2. **交互 > 阻力**：现代障碍要和目标/特殊宝石/布局**交互**逼出规划，不是变硬。
3. **障碍要"赚得席位"**：必须有独特交互，否则不加（反换皮硬标准）。
4. **视觉=机制对齐**：长什么样=怎么用，玩家直觉懂。
5. **难度锯齿**：难易交替，难关后给喘息。
6. **资源思维+cascade**：留出"造特殊宝石、设级联让关卡自己玩"的空间。
7. **AI bot 估难度 + 人定手感**：机器跑量，人把关 feel。

### 3.5 教学关
每出一个新道具(新障碍/特殊宝石/目标类型)，配**单独一关**就地教：环境压最简、新道具是唯一变量、几乎不会输。是难度曲线上的低谷。

## 4. Settings：一个关卡的完整可调维度（客体维度的操作化）

做关卡=调这些 settings。这是 levels.json 里一关的真实字段：

- **棋盘**：形状 · 尺寸 · 孔洞 · 出口/掉落点 · 重力方向
- **宝石**：颜色数 · 初始分布规则 · 生成器
- **障碍**：种类 · 位置 · 数量 · 层数/参数 · 与目标&特殊宝石的交互
- **目标**：类型 · 量 · 位置
- **资源**：步数 · 可用技能/道具及配额 · 预置特殊宝石
- **难度元数据**：序列角色 · 复杂度级 · 目标通关率 · 已解锁机制集
- **美学**：主题 · 配色 · 视觉焦点标记
- **时间**：预期单关时长

## 5. 做一关的流程（含约束传播——精髓）

### 5.1 十步流程
A 定坐标(① 读序列角色 ② 领复杂度预算) → B 出题(③ 选题眼 ④ 选地形原型 ⑤ 钉目标+布障碍 ⑥ 埋破局路径) → C 调校(⑦ 标难度系数 ⑧ 四重对齐) → D 验证(⑨〔机〕玩家模拟器 ⑩〔人〕试玩定手感)。

### 5.2 A→B 的传动轴 = "出题配方"
坐标不直接出题，先翻译成一张**配方**(机制额度/交互深度/目标档/地形档/松紧 S)；B 在配方约束内选。配方=复杂度阶梯的操作化。角色→配方有固定映射(教学/变奏/施压/波峰/喘息各一套)。

### 5.3 约束传播网 + 反向回退（流程的灵魂·不是随便搭）
每步的选择被上游**圈定候选子集**、再向下游传约束，自由度越往后越小：
```
坐标 ──过滤──▶ 题眼候选子集
题眼 ──锁定──▶ 地形类型(远征→沙漏 运输→带出口 围城→孤岛 隔断→分叉)
地形 ──锁定──▶ 目标&障碍的位置语法(沙漏→目标钉下游死水)
题眼+地形 ──逼出──▶ 破局路径(不是设计的，是被逼出来的)
破局路径 ──反推──▶ 棋子分布(颜色/空间) + 步数
```
**横向约束**：复杂度预算=全局红线(任一步超支→回退)；玩家模拟器=终检(难度不匹配→反向迭代)。
**实质**：流程难在"约束怎么一层层传、错了怎么弹回"，不在步骤顺序。

## 6. 整体设计：从零到 1000 关

**总览**：先用一次性投入建好「5 库 + 1 玩家模拟器」→ 排一条 1000 关难度曲线 → 每关走同一条流水线产出、机器验难度、人工抽检手感 → 上线数据再平衡。

### Phase 0 · 建 5 个库（地基，人主导）
题眼矩阵 · 地形原型库(~8，参数化+四位一体标签) · 障碍集(2 静+1-2 动，每个带交互) · 配方表 · 特殊宝石组合表(已有)。

### Phase 1 · 定玩家模拟器
输入关卡定义 + 玩家画像 → 输出首胜尝试分布/剩余步/失败原因/机制使用率/难度估计。先定规格，工程随后。

### Phase 2 · 手搓 20 关跑通流水线
照流程逐关做，验证"流水线能稳定产好关卡"，打磨库与配方。

### Phase 3 · 排 1000 关难度曲线（骨架）
一张表每关一行：角色/复杂度级/已解锁机制/主题组。编排：锯齿 + 复杂度阶梯 + 机制引入点(配教学关) + 主题成组换皮。

### Phase 4 · 批量产出 + 验证 + 运营
每关：读骨架→配方→取库实例化→玩家模拟器验→人工抽检→主题换皮。上线后看真实通关率，**数据再平衡**(⑥ 数据维度)。

**人机分工**：人=建库/出题创意/品味抽检；机=实例化/玩家模拟器/批量变奏。
**内容生命周期**(③ 时间维度)：1000 关不是一次做完，是持续产出 + 老关再平衡 + 活动关。

## 7. 速查：判断与禁令

- ✅ 每关一道题(有抽掉就塌的题眼)；视觉焦点=玩法焦点；一形四位一体。
- ✅ 难度靠"更需要想"(认知)，步数(执行)只微调；参数相对化(S/O/K/D)。
- ✅ 全程有情绪弧与爽感节奏(连锁/大招/绝杀)。
- ✅ 障碍极简、每个"赚得席位"(独特交互)；深度押在特殊宝石组合 + 地形/布局。
- ✅ 六维都回答(客体/主体/时间/感官/经济/数据)，不只做客体。
- ❌ 不做："消 X 个某色"目标 / 纯消除无障碍关 / 换皮重复关 / 引入新操作的障碍。
- ❌ 不照搬头部竞品的稀释+大段教学+换皮；不做章节系统、不做 RPG 战力养成。
- ❌ 不被旧 levels.json / 旧 overlay / 旧 C++"宝石排布"带偏。

# 04 CORE：消除（算法驱动的水果消除）★主线 Core★

> 本文档定义**主线 Core**:一款水果主题的消除游戏,吸收开心消消乐 / Candy Crush 的成熟机制,但**关卡由算法生成**(护城河,见 `09`)。
> 设计哲学对照 `01`,Meta 对接见 `02`,算法实现见 `09`,关卡设计视角见 `08`,技术接口见 `05`。

---

## 一、一句话定位

**同样的消除机制,别人的关卡是人工设计的、固定的、所有人一样的;我的关卡是算法生成的、无限的、个性化难度的、有最优解可竞技的。**

- 差异化**不在「元素比 Candy Crush 多」**(拼不过,也不该拼——它 30+ 障碍靠人工设计师驾驭无限组合,那是它的护城河、我的短板)。
- 差异化**在「同样的元素,我的关卡是『活』的(算法生成+个性化+可竞技),它的是『死』的(人工固定)」**。
- 少数元素 + 算法,打的是「动态 / 个性 / 无限」这张牌。

---

## 二、核心玩法（吸收自两大标杆，机制层，合法可抄）

> 法律边界(见对话结论):**机制(怎么玩)可放心吸收;表达(美术/名字/音乐/具体命名/具体关卡)必须 100% 原创**。下面是机制。

### 2.1 基础消除（low floor，人人秒懂）
- 滑动/交换相邻两个水果,**3 个同种类横竖相连即消除**。
- 消除后上方水果下落,顶部**随机补充**新水果(跟随行业:补充是随机的,见 `09` 第 0 节)。
- 补充可能形成新消除 → **连锁(cascade)**。

### 2.2 匹配维度：单维（种类）
- **消除按「水果种类」单维匹配**(3 个同种类即消)——这是消除的经典体验(开心消消乐/Candy Crush 都是单维),门槛最低、最顺手。
- (「种类 + 进化阶段」的两维匹配留给合成 Core,见 `03`;消除用单维,避免规则拧巴、门槛升高,也让算法更好驾驭。)

### 2.3 多连特效（high ceiling 的核心来源，可计算）
吸收自两大标杆的特效系统(机制层):
- **4 连(一行/一列)** → 生成「直线特效」水果(消整行/列)。
- **T / L 形** → 生成「爆炸特效」水果(炸周围一片)。
- **5 连** → 生成「彩球」水果(与任意水果交换,消全屏该种类)。
- **特效可组合**:两个特效一起消 → 更大连锁。

> 特效机制是「规则层」深度:人人面对同规则,**可计算**(算法能算「凑特效的最优布局」),且**放大「平庸 vs 最优」的差距**——贪心玩家不刻意凑特效,高手刻意凑 → 喂 `09` 的差距/投票筛选。这是 high ceiling 的实实在在的内容。

### 2.4 障碍系统（增加多样性，但严格受限）
吸收**少数规则清晰、状态确定**的障碍(机制层),由算法生成布局+验证有解(见 `09`):
- ✅ **冰块/糖霜(多层阻挡)**:相邻消除打破,N 层。(开心消消乐核心障碍)
- ✅ **锁**:消除含它才解锁。
- ✅ **果酱(裹住)**:相邻消除剥开。
- ✅ **底层目标(果冻类)**:消除覆盖到的格子算清掉一层。
- ⚠️ 蛋糕炸弹类(占多格+清场):可吸收 1 种,谨慎。

> **绝不吸收**:巧克力 / 黑巧克力(蔓延)、魔法搅拌器 / 巧克力喷泉(增殖生成新障碍)、传送带 / 加农炮 / 障碍嵌套。
> **理由**:这些「动态自我繁殖 / 复杂交互」的元素会让状态空间指数爆炸、求解失效(见 `09` 纪律 6),且是 Candy Crush 最依赖人工调的部分。**障碍种类受限,正是「算法可驾驭」与「靠人工驾驭无限复杂」的分界。**

### 2.5 战场切割（空间策略）
- 棋盘可被 WALL / 隔断切成**多个区域**,增加空间策略。
- **由算法生成 + 验证连通性**(防孤立死区,见 `09` 2.5 / FI2Pop 约束)。

### 2.6 关卡目标（通关条件，5-7 种，大多可计算）
吸收两大标杆的目标类型(总共就这几种,远少于障碍,且大多可计算——见 `09` 1.3):
- **得分**:达到目标分。
- **收集数量**:消除 N 个某种水果。
- **清障碍**:清掉 N 个冰块等(底层目标)。
- **限定步数**:上述目标在 M 步内达成。
- **混合**:上述组合(免费的多样性)。
- ⚠️ 掉落收集(运物到底部):可吸收,但下落路径让求解复杂些。
- (引流类较复杂,首发可不做。)

### 2.7 滚动/挖矿关卡（一种关卡形态）
吸收开心消消乐的「滚动挖矿」形态:
- 清掉当前可见区域 → 下方新内容滚上来,持续「挖」,收集够 N 个目标算赢。
- **注意**:滚动内容仍是**随机补充**(跟随行业);它是「关卡形态多样性」的补充,不是确定序列。
- 滚动关里**障碍更克制**(滚动已增加复杂度,别叠太多)。

## 三、Low Floor / High Ceiling 怎么体现

| | 体现 |
|---|---|
| **Low Floor** | 3 连消除人人秒懂;算法保证每关「有解、地板低」(休闲玩家能过);无角色/铭文也能玩(裸 Core 可运行,见 `09` 第 0 节) |
| **High Ceiling** | 凑多连特效/连锁需规划;算法保证每关「天花板高」(高手能拿远超地板的分);同题竞技比谁更接近最优解 |

> 关键:**地板低 + 天花板高,由 `09` 的「求解器组投票」保证**——低水平选民能过(地板)、高水平选民得分远超(天花板),差距大才是好关卡。

---

## 四、Core 与 Meta 的对接（资源流，见 `02`/`05`）

- **Core 产出 → Meta**:分数(消除/连锁/特效越优,分越高)→ 解锁抽卡机会 + 铭文碎片;关卡 seed → 同题竞技。
- **Meta 注入 → Core**:当前携带的「角色技能」(一个,一局用一次,第二次看广告)+「铭文加成」(一组)在本局生效。
- **关键**:Core 默认**无 Meta 也能玩**(守 low floor 生死线,也是 `09` 算法评估的基准)。Meta 是锦上添花。

---

## 五、算法在消除 Core 的应用（护城河，详见 `09`）

- **关卡 = 初始棋盘(含障碍/战场切割) + 目标 + 步数 + 补充随机分布**(因补充随机,无确定序列)。
- 算法对每个关卡:**生成**(构造/FI2Pop/MAP-Elites/CP/generate-and-test)→ **求解器组投票评估**(贪心~MCTS 代表不同玩家,多次模拟取期望)→ 得「有解性 + 难度 + LFHC + 节奏曲线」→ **筛选 + 打标签** → **三层调度 + 个性化推送**。
- 因补充随机,**求解主力 MCTS**(蒙特卡洛,处理随机期望),A* 仅小规模校准。
- **全程非黑箱**(构造/搜索/约束求解/MCTS/启发,不引入机器学习)。

## 七、注意（守住的边界）

1. **障碍严格受限**:只吸收规则清晰、状态确定的少数障碍;绝不碰蔓延/增殖型(否则算法失效)。
2. **多样性靠「目标 + 混合 + 算法生成布局」**,不靠「堆障碍元素」(后者是 Candy Crush 的护城河、你的死战场)。
3. **吸收机制,不吸收人工内容模式**:不复刻它的海量人工关卡、不照搬具体关卡/命名/美术(见法律边界)。
4. **守所有原则**:LFHC、不卖强度、角色一局一次且锦上添花、皮肤纯外观、算法藏底层。

# 猎场 · 看聊记 呈现切片 设计（仅"这一击"的 UI/交互）

日期：2026-05-17
范围：只锁**开天眼·看聊记这一块屏的呈现与交互形态** + 一个最小可达入口。概念已在 `2026-05-17-hunting-party-concept-design.zh.md` 锁定，本 spec 不重述/不推翻概念，只把"界面/呈现"那条被故意推迟的项补上。后端切片已交付（`Content.others_chat` 语料 + `PeekChat.peek` + `peek_smoke`，真相永不泄）。

---

## 一句话

被勾来的男人列一个表，点一个 → 一块**专屏**把"**他对你说的原话**"和"**他对别人说的原话**"上下并排——同一张嘴，两副面孔，玩家自己读出糖/料。不打分、不判定、无乙女装饰。这是抓住女性玩家的第一击，为情感冲击而做。

## 1. 后端小改（spec-safe，唯一后端改动）

`PeekChat.peek(man)` 当前返回 `{name, surface_claim, others_chat}`。并排需要他对你的原话，故再返回 `to_you_chat`：

- `to_you_chat` = `man.get("chat", [])` 的**浅拷贝**（与 `others_chat` 同一防御性拷贝纪律，防止把 Content 内部引用交给消费者）。
- 仍**绝不返回** `hidden_type` / `risk`。他对你说的是"表演面"不是真相——与已锁的"线索非答案 / 永不泄真相"纪律完全一致（`chat` 字段本就是他演给她看的）。
- 测试（追加进 `tests/test_peek_chat.gd`，不动既有测试）：①`peek()` 含 `to_you_chat`、形状为 `{from,text}` 行；②`to_you_chat` 与源 `man["chat"]` 是不同引用（`not is_same`）、值相等；③对三个男人 peek 仍 `not has("hidden_type")` 且 `not has("risk")`（回归守住核心不变量）。

## 2. 揭开屏（核心呈现）

竖屏 1170×2532，复用 Plan E 的 `Godot/ui/UiKit.gd` / `Theme.gd`（`screen()` 根 + 门控淡入、`panel/label/btn`、token 冷高级色板、字体对）。纯代码生成，无美术、无男性立绘。

- **顶部**：他的名字（`T.TITLE`/`T.ACCENT`）+ 一句冷标题（如"开天眼 · 他不演你的时候"）。无乙女装饰、无可爱 chrome。
- **竖向堆叠对比**（竖屏上下比左右好读）：
  - **上块面板「他对你说的」**：`to_you_chat` 渲成聊天气泡。他的话靠左气泡；偏暖一点点但仍在冷色板内——这是甜的表演面。
  - **中间·开天眼铰链**：一条冷金细线 + 小字（如"——他不演你的时候——"）。视觉铰链，"咯噔一下"就发生在这条线两侧的落差里。
  - **下块面板「他对别人说的」**：`others_chat` 每条一个气泡，带灰字接收者标签（直接用语料里的 `to`，如"另一个女生 · 1:14am"）。同一个人、另一张嘴。
- **永不判定**：不写"他是糖/料"、不打分、不结算、无"正确答案"提示。玩家自己读（已锁：台词即下注、线索非答案）。
- **底部**：返回键 → 回到列表入口。

气泡走**真实短信感**（这是情感冲击的来源），但皮肤用冷高级 token（深底、克制、单一香槟点缀、无圆角萌系装饰）：真实=内容，克制=皮。

## 3. 最小入口（能走到、能演示，不是整套派对）

一个独立可达的小流程，**自己一个场景**，不接 hub 外壳（外壳整合是后续切片）：

- **开天眼列表屏**：被勾来的男人（本切片暂用 `Content.men()`）列成可点的行：名字 + 一句中性引子（**不剧透** `surface`/`hidden_type`/糖料）。
- 点一行 → 揭开屏（第 2 节）。揭开屏返回 → 列表。
- 两态在同一个 `Godot/ui/Peek.gd` 里用一个状态切换（`list` / `reveal`），不引入复杂导航。

## 4. 文件与校验

- 改：`Godot/core/PeekChat.gd`（加 `to_you_chat`）、`Godot/tests/test_peek_chat.gd`（追加 3 断言/测试，不改既有）。
- 新：`Godot/ui/Peek.gd`（list/reveal 两态，复用 `UiKit`/`Theme`）、`Godot/scenes/Peek.tscn`（场景根挂 `Peek.gd`）。
- 改：`Godot/ui/Loc.gd`（**追加** zh 文案，不改既有、不重复 key）。
- 现有 hub / Plan-E / 引擎 / `play.gd` / `ui_smoke.gd` / `tuning.json` **一律不动**（纯新增屏 + 一处 spec-safe 后端加字段）。
- 校验：`godot --headless --quit` 退出 0；`run_tests.gd` → `RAN <67+新> tests, 0 failures`；`ui_smoke.gd` / `play.gd` 不受影响仍绿；窗口 sanity（`godot --path . res://scenes/Peek.tscn` 无脚本错误）+ 一个截图 helper 让用户能亲眼看这一击。

## 明确不在本 spec 内（后续各自独立切片）

- 接进 hub 外壳（常驻女主 + 底部按钮弹子页）。
- 派对的进场分支对话、H/S 两轴、读=下注的结算。
- 开天眼的稀缺/体力代价（"选中只能开几个"）。
- 男性人设 + 聊记 + 台词语料的量产（本切片只用现有 3 个男人的样本语料）。
- 约会侧"看十年后"。

本 spec 只锁"看聊记这一击长什么样、怎么走到"：并排原话对比专屏 + 最小列表入口，复用 Plan E UI 系统，冷高级/反乙女/纯代码，永不判定。作为后续外壳整合、对话、内容量产各自独立切片的呈现地基。

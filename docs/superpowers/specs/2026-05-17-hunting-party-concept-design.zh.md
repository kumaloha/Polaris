# 猎场 · 派对 概念定稿（仅概念，不含界面/数值/代码）

日期：2026-05-17
范围：定义"派对在这个游戏里到底是什么"。**派对是一个分支对话游戏**——这是地基，不是后期资料包。本 spec 锁概念模型；界面呈现、数值调参、代码结构、语料的具体量产是后续各自独立的 spec/阶段。

---

## 一句话

**派对是一个分支对话游戏，是建立关系的场，不是能输赢的关卡。** 你那一帖在社媒勾来一小批男人；注意力稀缺，搭不过来所有人。你逐个和他们对话——他说台词，你从几句回复里挑一句——在对话里读他、推关系。约会/结算只是去兑现你在派对对话里建起来的东西。

## 派对 = 分支对话游戏

- **一个男人 = 一条对话线。** 派对里你能开/续几条对话线，受注意力稀缺限制（沿用已锁定 `party` 调参：回合数、注意力池），搭不过来所有人，深聊更贵。
- **他说台词。糖 vs 料藏在他"怎么说"里。** 表面≠真相：满嘴甜话/殷勤可能是糖（虚），话冷/话少可能是真料。早期台词最模糊，越聊越能分辨。开天眼/聊天记录 = 模糊的早期前情，给线索不给答案。
- **你选台词。** 每一轮他说完，你从几个回复里挑一句。回复的**性质**就是 Control 姿态（沿用已锁定 B 后果模型）：
  - **追**型回复：你更主动/更贴 → 推 **S** 上 → 掉控制、吃 chase 罚。
  - **挣·挡**型回复：稳住/让他来 → 让 **H** 往你这边上 → 得控制、earn 收益。
- **读他 = 解读他的台词，下注就是你选哪句。** 选对——即你的回复匹配他**真实的隐藏态**——就把轴推对；读错人/选错句，就把轴推歪（典型：把糖当料，对一个虚的人选了追，S 飙、控制崩）。

## 关系 = 两根好感度轴

一个男人和你的关系不是命名类型，是一个二维坐标，由对话路径推出来：

- **轴 H：他对你的好感**——**隐的，会误读**。表面≠真相（糖 vs 料）。只能从他台词里估，开天眼只给模糊早期线索。读准 H 就是玩家的 alpha，判断力在这里长。
- **轴 S：你对他的好感**——你自己的投入，基本是显的（你知道自己选了多少追型回复、在谁身上花了几轮）。

| 象限 | 含义 |
|---|---|
| H 高 · S 低 | 他在追你/殷勤——**前提是他的"高"是真料不是糖**，否则是陷阱。有利杠杆位。 |
| S 高 · H 低 | 你在倒贴/掉控制——**最坏象限**；若他"看着高"其实是糖、你还追，最惨。 |
| 双高 | 相互、牢固。 |
| 双低 | 还没建立/无。 |

**误读 = 误判 H 的真值**（不是 S）。一夜里你以为自己在某象限，可能其实在另一个。

## 进场时男人的差异（隐，要从对话里读）

同一批被勾来的男人，**H 起点与"可推性"天生不同**：

- 有人 H 起点高、台词热（殷勤）——但要从话里分清真料的高 vs 糖的高。
- 有人 H 中、话不冷不热，**得多聊几轮**才推得动（深加深=花更多稀缺注意力）。
- 有人压根不想屌你（H 锁低、话敷衍）——或"冷"是装的、其实是真料。

## 读是隐式的：台词就是下注

派对**没有**显式"判定他是料/糖"的动作。你选的台词就是唯一的下注；读对了你自然把轴推对了。**判断力权益 = 一夜结束你建起的关系位置质量**——读对的人，关系自然落在好象限；读错的人，注意力烧光、轴推歪。不单独给判断打分、不做答题。

## 稀缺与产出

- **稀缺**：能维持的对话线数 + 每条线的轮数有限。深聊一个人 = 占掉更多稀缺注意力，开/续别的线就少。
- **一夜产出**：一组对话过的男人，各自停在某个 (H, S) 坐标（由对话路径决定）+ 你对他们的隐含判断（藏在你推轴推得对不对里）。这组东西带进约会/结算兑现，沉淀成净值——读对=判断力权益，读错=在错的人身上烧光了稀缺注意力。

## 核心内容依赖（模型在此锁定，量产是后续阶段）

对话是派对本身，所以这些是**核心、不是点缀**——本 spec 锁定它们的**模型形状**，具体撰写/量产放到后续独立的内容阶段：

- **男性人设库**：每个 persona 有隐藏的 (H 起点、可推性、料/糖底色)。
- **台词语料/对话树**：每个 persona 一套分支对话；他的台词要能"用怎么说来藏料/糖"；玩家每轮的回复选项要能映射到 追 / 挣·挡 等 Control 姿态；早期模糊、后期可分辨的递进。
- 这套语料量很大，**单独立项量产**；但其结构（人设字段、对话树形状、回复→Control 映射、糖/料话术规范）由本概念约束。

## 明确不在本 spec 内（后续各自独立）

- 派对的**界面/呈现**（对话怎么排版、台词怎么点、注意力/轴怎么可视化）。
- 具体**数值调参**（H/S 步长、读错的判定、深聊成本曲线、对话线上限等）。
- **代码结构 / 引擎改动**。
- 男性人设 + 台词语料的**具体撰写与量产**（模型形状在上一节已锁，内容本身后续立项）。

本 spec 只锁"派对是什么"：一个分支对话游戏，关系=两轴好感度，读=隐式（台词即下注），对话语料为核心依赖。作为后续界面、机制数值、内容量产各自独立 spec 的共同地基。

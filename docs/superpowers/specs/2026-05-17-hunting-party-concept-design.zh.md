# 猎场 · 派对 概念定稿（仅概念，不含实现/界面）

日期：2026-05-17
范围：**只定义"派对在这个游戏里到底是什么"**。不含界面布局、玩法数值、代码结构、内容资料包——那些是后续各自独立的 spec/计划。

---

## 一句话

**派对是一个"建立关系"的场，不是一个能输赢的关卡。** 你那一帖在社媒勾来一小批男人；注意力稀缺，搭不过来所有人。你在这里挑谁值得花、肯花几轮、把关系推到什么位置——而每个男人对你的真实态度是隐的、会被你读错。

派对不产出结果，它产出**关系位置 + 你对这些人的判断**。约会/结算只是去兑现你在派对里建起来的东西。

## 关系 = 两根好感度轴（核心模型）

一个男人和你之间的关系，不是一个命名类型，而是一个二维坐标：

- **轴 H：他对你的好感**——**隐的，会误读**。表面≠真相：表面殷勤可能是糖（H 实际低），表面冷可能是真料（H 实际高）。开天眼 / 聊天记录只给**模糊的早期线索**。读准 H 就是玩家的 alpha，判断力在这里长。
- **轴 S：你对他的好感**——这根是你自己的投入与选择，基本是显的（你知道自己在谁身上花了多少）。

坐标位置定义"关系形状"：

| 象限 | 含义 |
|---|---|
| H 高 · S 低 | 他在追你 / 殷勤——**前提是他的"高"是真料，不是糖**。这是有利杠杆位。 |
| S 高 · H 低 | 你在倒贴 / 掉控制——**最坏象限**；若他那"看着高"其实是糖、你还往上贴，最惨。 |
| 双高 | 相互、牢固关系。 |
| 双低 | 还没建立 / 无。 |

**误读 = 误判 H 的真值**（不是 S）。一夜里你以为自己处在某个象限，可能其实在另一个。

## 进场时男人的差异

同一批被勾来的男人，**起点 H 与"可推性"天生不同**（这部分对玩家是隐的，要读）：

- 有人 H 起点高、好搭（殷勤）——但要分清是真料的高还是糖的高。
- 有人 H 起点中、**得多聊几轮**才推得动（深加深需要花更多稀缺注意力）。
- 有人压根不想屌你（H 锁低）——或"冷"是装的、其实是真料。

## Control（追 / 挣）如何作用在两轴上

每次互动玩家选 Control 姿态（沿用已锁定的 B 后果模型）：

- **追**：把 **S** 往上推（你更投入 / 主动贴）——掉控制、吃 chase 罚；把你推向 S 高象限。
- **挣**：让 **H** 往你这边上来（他更投入）——得控制、earn 收益；把关系推向 H 高 / 双高。

派对里的每一回合，本质就是"用一份稀缺注意力，在某个男人身上推动 H 或 S 一格，且你对 H 的真值始终是估的"。

## 稀缺与产出

- **稀缺**：注意力有限（沿用已锁定 `party` 调参：回合数、注意力池）。搭不过来所有人，深加深更贵。
- **一夜产出**：一组男人，各自停在某个 (H, S) 坐标 + **你对每个人 H 真值的信念（可能是错的）**。这组东西带进约会/结算兑现，沉淀成净值——读对 = 判断力权益，读错 = 在错的人身上烧光了稀缺注意力。

## 明确不在本 spec 内（后续独立阶段）

- **男性人设 + 对话资料包**（量很大，单独立项：人设库、信号/聊天文本、糖vs料的话术样本）。
- 派对的**界面/呈现**（怎么摆、怎么点）。
- 具体**数值调参**（H/S 步长、读错概率、深加深成本曲线等）。
- 代码结构 / 引擎改动。

本 spec 只锁"派对是什么"与"两轴关系模型"，作为后续界面、机制数值、内容包各自独立 spec 的共同地基。

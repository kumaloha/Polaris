# 猎场 · 派对 概念定稿（仅概念，不含界面/数值/代码）

日期：2026-05-17
范围：定义"派对在这个游戏里到底是什么"。**派对是一个分支对话游戏**，赛前有开天眼（看他跟别人的聊记）。本 spec 锁概念模型；界面呈现、数值调参、代码结构、语料的具体量产是后续各自独立的 spec/阶段。

---

## 一句话

**派对是一个分支对话游戏，是建立关系的场，不是能输赢的关卡。** 你那一帖在社媒勾来一小批男人；赛前可以对选中的人开天眼、偷看他跟别人怎么聊；进场后逐个和他们对话——他说台词，你从几句回复里挑一句——在对话里读他、推关系。约会/结算只是去兑现你在派对里建起来的东西。

## 派对 = 分支对话游戏

- **一个男人 = 一条对话线。** 派对里你能开/续几条对话线，受注意力稀缺限制（沿用已锁定 `party` 调参：回合数、注意力池），搭不过来所有人，深聊更贵。
- **他说台词。糖 vs 料藏在他"怎么说"里。** 表面≠真相：满嘴甜话/殷勤可能是糖（虚），话冷/话少可能是真料。早期台词最模糊，越聊越能分辨。
- **你选台词。** 每一轮他说完，你从几个回复里挑一句。回复的**性质**就是 Control 姿态（沿用已锁定 B 后果模型）：
  - **追**型回复：你更主动/更贴 → 推 **S** 上 → 掉控制、吃 chase 罚。
  - **挣·挡**型回复：稳住/让他来 → 让 **H** 往你这边上 → 得控制、earn 收益。
- **读他 = 解读他的台词，下注就是你选哪句。** 选对——即你的回复匹配他**真实的隐藏态**——就把轴推对；读错人/选错句，就把轴推歪（典型：把糖当料，对一个虚的人选了追，S 飙、控制崩）。

## 派对前的开天眼：看他跟别人的聊记

进场对话之前，你可以对**选中的**男人开天眼。

- **看的是他跟别人的聊天记录，不是他对你的表演。** 他给你的是演的；他对别的女生/别人怎么说话，才是不演你时的真面目——这是最强的"戳穿表面≠真相"的赛前情报。
- **人设是其次，聊记才是最重要的。** profile/标签只是背景；判断真值靠读他那段聊记的话术（糖 vs 料藏在他怎么对别人说）。
- **它是线索不是答案。** 聊记把对他真实 H / 料糖的估计往更准推一点，但仍要你读、仍会读错；进场对话里你拿这个赛前假设去试、去用。
- **稀缺、有代价。** "选中"=只能挑几个开（沿用已锁定的开天眼/体力代价；搭不过来所有人，赛前情报也搭不过来所有人）。
- 注：这是**派对前**的开天眼。另一种开天眼（"看十年后"）是约会深入到一定程度才解锁的，属约会侧，不在本 spec。

## 首要钩子（设计原则）

**看他跟别人的聊记，是抓住女性玩家的第一个点**，是本作开天眼幻想的情感核心——终于看见他不演你时的真面目。设计后果：

- 这一刻**不是工具性情报面板**，要为**情感冲击**而做：真面目被揭穿的爽 / 痛必须落地（他对你甜、对别人原形毕露；或他对你冷、对别人也一样真诚）。
- 它是**优先打磨的垂直切片**——先做这一块、先把它做到位，再铺其余派对。
- 语料与呈现都服务这一击：聊记文本要写得"读出来心里咯噔一下"，不是干巴巴的标签。

## 关系 = 两根好感度轴

一个男人和你的关系不是命名类型，是一个二维坐标，由对话路径推出来：

- **轴 H：他对你的好感**——**隐的，会误读**。表面≠真相（糖 vs 料）。赛前开天眼（他跟别人的聊记）给较强但仍模糊的线索，进场只能从他对你的台词里继续估。读准 H 就是玩家的 alpha，判断力在这里长。
- **轴 S：你对他的好感**——你自己的投入，基本是显的（你知道自己选了多少追型回复、在谁身上花了几轮）。

| 象限 | 含义 |
|---|---|
| H 高 · S 低 | 他在追你/殷勤——**前提是他的"高"是真料不是糖**，否则是陷阱。有利杠杆位。 |
| S 高 · H 低 | 你在倒贴/掉控制——**最坏象限**；若他"看着高"其实是糖、你还追，最惨。 |
| 双高 | 相互、牢固。 |
| 双低 | 还没建立/无。 |

**误读 = 误判 H 的真值**（不是 S）。一夜里你以为自己在某象限，可能其实在另一个。

## 进场时男人的差异（隐，要从开天眼+对话里读）

同一批被勾来的男人，**H 起点与"可推性"天生不同**：

- 有人 H 起点高、台词热（殷勤）——但要从话里分清真料的高 vs 糖的高（赛前看他对别人是不是也这样，是最好的破绽）。
- 有人 H 中、话不冷不热，**得多聊几轮**才推得动（深加深=花更多稀缺注意力）。
- 有人压根不想屌你（H 锁低、话敷衍）——或"冷"是装的、其实是真料。

## 读是隐式的：台词就是下注

派对**没有**显式"判定他是料/糖"的动作。开天眼给你赛前假设，进场后你选的台词就是唯一的下注；读对了你自然把轴推对了。**判断力权益 = 一夜结束你建起的关系位置质量**——读对的人，关系自然落在好象限；读错的人，注意力烧光、轴推歪。不单独给判断打分、不做答题。

## 稀缺与产出

- **稀缺**：赛前开天眼能开的人数有限；进场能维持的对话线数 + 每条线的轮数有限。深聊/多开天眼一个人 = 占掉更多稀缺资源，别的就少。
- **一夜产出**：一组对话过的男人，各自停在某个 (H, S) 坐标（由开天眼假设 + 对话路径决定）+ 你对他们的隐含判断（藏在你推轴推得对不对里）。这组东西带进约会/结算兑现，沉淀成净值——读对=判断力权益，读错=在错的人身上烧光了稀缺资源。

## 核心内容依赖（模型在此锁定，量产是后续阶段）

对话是派对本身，所以这些是**核心、不是点缀**——本 spec 锁定它们的**模型形状**，具体撰写/量产放到后续独立的内容阶段：

- **男性人设库**：每个 persona 有隐藏的 (H 起点、可推性、料/糖底色)。profile 信息其次。
- **赛前聊记语料**：每个 persona 一段"他跟别人的聊天记录"文本，糖/料藏在他对别人怎么说里——这是开天眼看的东西，是核心情报内容。
- **台词语料/对话树**：每个 persona 一套分支对话；他的台词要能"用怎么说来藏料/糖"；玩家每轮的回复选项要能映射到 追 / 挣·挡 等 Control 姿态；早期模糊、后期可分辨的递进。
- 这套内容量很大，**单独立项量产**；但其结构（人设字段、赛前聊记形状、对话树形状、回复→Control 映射、糖/料话术规范）由本概念约束。

## 明确不在本 spec 内（后续各自独立）

- 派对的**界面/呈现**（开天眼聊记怎么看、对话怎么排版、注意力/轴怎么可视化）。
- 具体**数值调参**（H/S 步长、读错的判定、深聊/开天眼成本、对话线与开天眼名额上限等）。
- **代码结构 / 引擎改动**。
- 男性人设 + 赛前聊记 + 台词语料的**具体撰写与量产**（模型形状在上一节已锁，内容本身后续立项）。
- 约会侧的"看十年后"开天眼。

本 spec 只锁"派对是什么"：一个分支对话游戏，赛前开天眼看他跟别人的聊记（**这是抓住女性玩家的首要钩子、优先垂直切片**），关系=两轴好感度，读=隐式（台词即下注），人设/聊记/对话语料为核心依赖。作为后续界面、机制数值、内容量产各自独立 spec 的共同地基。

# Hunting（猎场）Hub IA 设计 v3：竖屏 hub 外壳 + 既有 carry 引擎为灵魂

状态：经会话共识锁定的 IA 主稿。**不改动游戏机制内核**——内核仍以 `docs/superpowers/specs/2026-05-17-hunting-carry-design.zh.md` 为准（carry 账本 / Control 你追还是让他挣 / B 模型 / 开天眼 / 净值 / 镜像 / 软 ruin / 可接受性脊柱）。本稿只定**信息架构与表现层**：把已跑通的引擎从"6 屏线性夜循环"重壳为"竖屏 hub + 底部导航"，并补一个净新支柱（社交媒体）。

取代：现有 `ui/UIController.gd` + `ui/Screens.gd` 的线性 6 屏流的 **IA**（引擎、`SeasonFlow` 交互 API、40 测试、carry 机制全部复用/扩展，不推翻）。

数值一律占位，标 `[TO-TUNE]`。

---

## 0. 一句话

它仍是"Amy 姐培训班的游戏版 / carry 读人引擎"，只是从线性夜循环改成**像真实手游一样的竖屏 hub**：首页是"你现在是谁"，底部 tab 进入各功能面，所有面都把数据喂回同一套 carry/净值核心。

## 1. 可接受性脊柱（不变，统辖所有文案）

沿用 carry-spec §1 七条：防御框架 > 获取框架；telos=她自己不是男人；开天眼=招牌；永不教只让她发现+验证；Amy 姐分散进闺蜜（Nina 口吻）；露骨处上皮肤；分层可接受性。**集卡是"她的判断力与社交资本"，绝不是抽帅哥/养成。**

## 2. 架构

- **灵魂 = 既有引擎**：`GameState`（含净值资产负债表）、`ControlEngine`、`FirstEye`、`PartyEncounter`、`Book`、`FutureEye`、`Girlfriends`、`SoftRuin`、`SeasonFlow`（含交互式子步骤 API）。
- **壳 = 竖屏 hub**：首页 + 底部导航 + 各功能面，逐像素视觉后续照参考 app 重蒙皮（本稿不锁视觉，沿用冷感金代码生成基线 + Loc 中文层）。
- **时间模型 = 软日循环（结构抉择 a，已锁）**：hub 可自由逛（随便点 tab），但**去派对并结算约会会推进一个"夜"**；`Energy` 是节流阀（机会成本，首页精力条）。映射现有 `SeasonFlow`：一个"夜" = 现有 `begin_night → 派对 → resolve_after → finish_night`；每 ~6 夜 `settle()` 周结算；~18 夜 `close_season()` 赛季落幕。自由逛不强制顺序，但精力 + 夜推进给节奏。

## 3. 首页

- 中央：玩家游戏形象 = 当前**装扮 + 人设**的可视；
- 数值：`Day/S/W`、**精力条（机会成本，最显眼）**、净值微提示、Charm/Position(Standing)/Control 氛围化呈现（非表格，沿用 HUD 风格）；
- 底部导航 tab：自我提升 / 社交媒体 / 派对 / 约会 / 资产（5 入口；集卡已并入资产——在 carry/资产负债表内核里，正确读懂的男人=判断力权益=净值资产，故集卡作为资产页的"持仓"与资产负债表合并；开天眼不是 tab，是嵌在派对前与约会内的功能）。
- 首页本身是"镜子"：你为认同 decumulate 自己时，这里第一眼能感到（人设漂移/精力枯）。

## 4. 七个面

### 4.1 自我提升（换壳扩展 Ready Room）

人设 / 装扮 / 运动，三者都是**精力下的机会成本分配**（沿用 carry-spec §8.1：4 张自投卡 → 耐久轨，机会成本是深度本身）：
- 装扮（Outfit）：今晚形象，决定 Charm 与"发图素材"；
- 运动（Solo Reset 家族）：Energy/状态复位；
- 人设（Persona）：战略身份，路径依赖（切换有代价，世界会记住），改变哪些读牌线/发帖线生效（Rare Girl/Soft Sun/Power Darling 扩展）。
- 产出一个"今晚 build" → 直接喂 4.2 社交媒体的发帖。
- 引擎映射：`Content.self_investments()` + `personas()` + `begin_night(self_invest,persona)`；装扮/运动是其换皮+扩展（**净新内容**：装扮项、运动项需进 `Content` + `tuning.json`，纯追加不破 40 测试）。

### 4.2 社交媒体（**净新核心支柱** — 漏斗顶）

**作用：吸引男性搭讪 + 扩展闺蜜圈。** 即"换装后拍图发一发"。

机制（A 已锁：社媒玩法本身就是游戏，不另做街机小游戏）：
- **发帖 = 一个 Control/B 抉择。** 内容是当前 build 的表达；选发什么/怎么发：
  - 克制 / 稀缺叙事 → 触达少但**高档**，Standing 慢复利，Control 守住；
  - 为认同 / 追流量 / 蹭热点狂发 → 触达**多但柠檬**（负 carry/糖男占比高，柠檬市场），即时多巴胺，**为验证而发 → 触发镜像**（"你也是别人的红旗"，发生在公开场）。
  - 复用 `ControlEngine` 的 chase/earn + B 模型同一套语义（发帖是它的新入口，不是新机制）。
- **产出 1：男人池。** 谁滑进私信/谁会在你下一场派对出现，由发帖质量×build×Standing 决定（这是"对你自己的 First Eye"的反面：你在被定价）。男人池流入 `Book`/派对待读列表。
- **产出 2：闺蜜线索。** 高 Standing/对味内容 → 解锁/升温闺蜜（`Girlfriends.adjust`），扩张进高端派对的信用阶梯。
- **评论/私信 = 低风险高频读人**（brief 的"高频鉴定"面）：滑进来的男人、假人设、捧场闺蜜——快速读、回应/筛/拉黑，喂**案底（判断力权益）**。和派对读牌同一套技能，节奏快、代价低。
- 引擎接线（**净新**）：`SeasonFlow` 加交互方法 `compose_post(build, posture) → {inbound_men:[], gf_leads:[], control_delta, standing_delta, mirror?}`；`posture` ∈ scarce/validation（chase/earn 映射）；产出写进当夜派对男人池与 `Girlfriends`。评论/私信读人产出写 `state.dossier`（**顺带补上一直延后的"案底生产端"**）。
- **时序**：`compose_post` 是**夜内、派对前**的一步（自我提升之后、派对之前），它本身**不推进"夜"**（`day+1` 仍只由 `finish_night()` 做）；一个夜可发一次帖（多发/重发策略留 `[TO-TUNE]`，默认每夜一次）。

### 4.3 派对（换壳融合 PartyEncounter + First Eye + 闺蜜成长）

- **有男有女。** 男人 = 社媒漏斗产出的池子；女人 = 闺蜜线索。
- 赛前：**开天眼·聊天记录**（First Eye：表面 claims + 聊天证据 + 案底自动打标，不漏 hidden_type）。
- 场内：读牌循环（试探/界限/社证/离场），**自己选和谁互动**：投男人 = 开/管 carry 仓；投女人 = 升温闺蜜、扩网络。
- 派对档位仍由 Standing/闺蜜网络门控（取消独立"闺蜜局"屏后，网络靠 4.2 社媒 + 这里和女生互动来扩）。
- 引擎映射：`available_parties()`/`choose_party()`/`party_men()`/`start_party()`/`record_party_action()`，扩展为"男人池来自社媒漏斗 + 女人=gf 线索"。

### 4.4 约会（换壳扩展 After-Party + Future Eye 门控）

- 派对/私信遇见的人，进入约会面：**是否约 / 怎么约 / 去哪玩什么**。
- = `Book` 的 Date/Observe/Test/Cut 决策 + 时间演化（持仓变异、债主、幻想债）。
- **开天眼·十年** = 一段关系**约会深入到一定程度后才解锁**（按用户定义；映射 `FutureEye.resolve`，门控条件 = 该仓投入深度阈值 `[TO-TUNE]`，而非任意 Date 即看）。十年帧 = 被验证的未来（关键帧），写 `state.keyframes`。
- 镜像在低 Control / 资不抵债时点燃（沿用现有逻辑）。

### 4.5 开天眼（功能，非 tab）

两处出现，沿用 spec 与用户定义：① **聊天记录视图 = 派对前**（嵌在 4.3 First Eye）；② **十年关键帧 = 约会深入后解锁**（嵌在 4.4）。真相一路在迹象里，开天眼=回报确认+打分+镜像，不是魔法揭真相（carry-spec §6 不变）。

### 4.6 资产（合并：净值资产负债表 + 集卡持仓）

- 顶部：资产负债表——资产（Standing/案底/关键帧）− 负债（幻想债）= 净值；NET WORTH 大字显眼；周 `settle()` mark；赛季 `close_season()` 战报卡（可分享）。
- 底部（持仓/集卡）：三类卡——**男人案底**（判断力权益）；**闺蜜网络**（社交资本/信用阶梯）；**挣来的关键帧**（被验证的未来）。**挣来非抽**，这是"她的判断力与社交资本"的可视化，不是奖杯墙。
- 设计逻辑：在 carry/资产负债表内核里，正确读懂的男人=判断力权益=净值资产，集卡就是资产的"持仓明细"，两者合一。
- 引擎映射：`GameState.net_worth()`/`snapshot()`/`SeasonFlow.settle()`/`close_season()` + `state.dossier` + `Girlfriends._warmth` + `state.keyframes`。

## 5. 软日循环（怎么穿起 hub）

开 App → 首页（看 state/精力）。一个"夜"的自然路径（不强制，自由逛，但精力+夜推进给形状）：
**自我提升**(定 build，花精力) → **社交媒体**(发帖→生成男人池+闺蜜线索) → **派对**(读/选男人+升闺蜜) → **约会**(对整本书决策；深入→可能解锁十年) → 时间推进 → 周期性看**资产清单**结算 + 镜像。映射 `SeasonFlow`：`begin_night`(自我提升) → `compose_post`(社媒，净新) → 派对交互 → `resolve_after`(约会) → `finish_night` → `settle`/`close_season`。

## 6. 这次改变/丢弃了什么

- **丢弃**：独立"闺蜜局"屏 + 现有线性 6 屏 `UIController` 流的 IA（引擎不丢）。
- **闺蜜圈扩张改由**：社交媒体经营 + 派对里和女生互动。
- **必须补**：`state.dossier` 的"生产端"（一直延后的口子）——由 4.2 评论/私信读人 + 派对读对来写。
- **新增内容数据**：装扮项、运动项、社媒发帖姿态/内容类型、男人池生成规则、评论/私信读人样本 —— 全部进 `Content` + `tuning.json`，纯追加，不改既有 40 测试断言。

## 7. 净新 vs 换壳清单

- **换壳现有引擎（低风险）**：首页、自我提升、派对、约会、集卡、资产清单的 UI；复用 `SeasonFlow` 交互 API。
- **净新引擎追加（需 TDD）**：社交媒体支柱（`compose_post` 漏斗 + 评论/私信读人 + 男人池生成 + gf 线索 + 案底生产端）；装扮/运动作为独立 `Content` 机制；女人作为派对参与者→闺蜜成长。
- **净新 UI**：竖屏 hub 导航壳（首页 + 底部 tab + 面间路由），替换线性 `UIController`/`Screens` 的结构。

## 8. 分阶段实现分解（供 writing-plans 拆成多个计划）

本 IA 较大，建议拆成 3 个顺序子计划，每个产出可测产物：

1. **Plan A — 引擎追加（TDD，headless）**：`SeasonFlow.compose_post` 漏斗 + 男人池生成 + gf 线索 + 评论/私信读人 + **案底生产端**；装扮/运动/发帖内容进 `Content`+`tuning.json`。门：现有 40 测试不变 + 新测试覆盖漏斗/案底/姿态(scarce/validation)语义；`play.gd`/批处理不破。
2. **Plan B — Hub 外壳 UI**：首页 + 底部导航 + 路由（替换线性 `UIController`/`Screens` 结构，保留可复用的 `Theme`/`Loc`/`_btn/_label` 原语 + 既有屏作为面内容重组）。门：headless boot + UI smoke 走通 hub 一夜。
3. **Plan C — 社交媒体面 + 集卡/资产清单面**：把 Plan A 的漏斗接进 hub 面；集卡浏览 Dossier/网络/关键帧；资产清单 equity 曲线。门：完整一夜含发帖→派对→约会→结算可点通，中文。

每个子计划单独 spec→plan→执行（subagent-driven），不一次性巨型化。

## 9. 验收标准

- 首页能在 3 秒内让人看出"这是一款游戏不是后台"，有形象/精力条/明确下一步。
- 玩家能复述：我发的帖怎么决定了来搭讪我的是什么货色；我为什么没追那个糖男；闺蜜怎么把我带进更高的局；十年帧为什么要约会够深才看。
- 社交媒体的发帖**确实**改变男人池质量与 Standing；为验证狂发**确实**会触发镜像。
- 集卡读起来是"我的判断力/网络"，不是奖杯墙；资产清单能判断这赛季净值走向。
- 全程可接受性脊柱过尺（防御性、不"钓凯子"、她是资产）；引擎 40 测试始终绿、纯追加。

## 10. 未决 / 延后 / TO-TUNE

- 全部数值平衡（漏斗触达系数、姿态对池质量的影响、十年解锁深度阈值、装扮/运动效果、案底情报折扣强度）`[TO-TUNE]`。
- 社媒"内容类型"具体清单与文案（Plan C 内容生产）。
- 派对中"和女生互动→闺蜜成长"的具体动作集（Plan A 设计时定，最小集即可）。

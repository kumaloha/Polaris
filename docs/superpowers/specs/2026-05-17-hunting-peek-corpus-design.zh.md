# 猎场 · 看聊记语料扩充 设计（纯增量内容，模型已锁不变）

日期：2026-05-17
范围：只把 `Godot/core/Content.gd` 的 `men()` 从 3 人扩到 **12 人**，每人聊记写厚，让"看聊记"这一击能被反复地、有变化地体验。**数据模型已锁，不改**；Peek 屏 / PeekChat / 引擎 / hub / 对话/轴/成本 全不碰。概念以 `2026-05-17-hunting-party-concept-design.zh.md` 为准（糖vs料、表面≠真相、开天眼=线索非答案、永不判定、"心里咯噔一下"、冷高级反乙女）。

---

## 一句话

把男人从 3 个加到 12 个（**只追加，不改名不重排既有 adrian/evan/leo 的 id**），每人按已锁 schema 写，`chat`（对你）卖表面、`others_chat`（对别人）让真相漏出来——量够大、伪装够多样，让玩家在 Peek 屏里反复练"读人"。

## 锁定不变（不在本 spec 讨论）

- schema 每人：`id, name, hidden_type∈{resource,high_sugar,growth}, surface, energy_cost, risk, opportunity, chat:[{from,text}], others_chat:[{to,text}]`。
- `hidden_type` 仍只此 3 类——**不新增真相类型**（那会动概念/引擎/测试）。
- `peek()` 永不返回 `hidden_type`/`risk`；UI 永不判定糖/料。这些已实现且测试锁住，不动。

## 数量与厚度

- 总 **12 人**：保留现 3（adrian 真资源/不装；evan 真糖/装成长=痛；leo 真料/装大佬=爽），**新增 9**。
- 每人厚度：`chat`（他对你）**2–3 句**，足够卖出 `surface` 的表面印象；`others_chat`（他对别人）**4–6 句**，多对象（另一个女生/前任/兄弟/同事/群聊/家人…），真相从"他对别人怎么说的方式"漏出来。
- `risk`/`opportunity` 一句到位，沿用现有简洁口吻（仅背景，UI 不显，但保持 schema 完整与引擎兼容）。

## 伪装覆盖矩阵（设计实质：用 3 真相类，变 surface↔truth 错位，教"读"）

新增 9 人覆盖以下格，多个实例、各自不同味（不要复制感）：

- **真糖伪装（痛，多实例不同味）**：糖装资源（晒预定/"我安排好了"，对别人全面子工程没下文）、糖装成长（"懂你、有深度"，对别人是回收的深沉话术、表演式脆弱）、糖装大佬（高调强势，对别人虚张声势、回避兑现）。
- **真料伪装（爽，多实例不同味）**：料装糖（对你油腻甜话，对别人具体靠谱有边界——会被误当渣男丢掉的那种）、成长装糖（对你贫嘴，对别人安静自省真在改）、料装冷（对你对别人都冷淡干脆但事事兑现——教"冷≠糖，料可以不浪漫"）。
- **一致样本（校准用）**：再补 1–2 个表里一致的（真资源/资源、真成长/成长），让玩家校准"不是所有人都在装"。
- 至少 **5 个 disguised**（surface≠hidden_type），且痛、爽两个方向都各有多个不同味实例。

## 文案标准（硬要求）

- 冷高级、反乙女、防御性、**具体**：时间戳、"另一个女生/前任/群聊"、复制粘贴的同一句、话冷但真、回避兑现的话术——沿用现有 adrian/evan/leo 的质感与英文口吻（项目语料现为英文，保持一致）。
- 真相必须**藏在"他对别人怎么说"里**，不靠标签；`chat` 对她要可信地卖出 `surface`，两者落差就是"咯噔一下"。
- 名字写实、多样、无乙女/二次元味。

## 校验

- 现有 `Godot/tests/test_peek_chat.gd` 完整性测试自动覆盖新人（每人 `others_chat`≥2、≥2 disguised、`peek` 不泄真相、distinct copy）。
- **追加**断言：`Content.men().size() >= 12`；存在覆盖关键 surface≠truth 错位格的若干已知 id（点名抽查 2–3 个新人的 hidden_type/surface 组合，像现有 `test_known_reveal_cases_present`）。
- 只**追加**男人，不改名/不重排既有 id；如有按 id 取人的测试或 `parties()` 引用，保持既有 id 不变即不受影响；任何对"男人数量"的硬编码断言相应调整。
- 全套测试保持 `0 failures`（数量上升，断言写成"≥"而非硬编码）；`peek_ui_smoke` 仍 OK（其 `men=%d` 仅打印不断言具体值）；`ui_smoke`/`play.gd` 不受影响（引擎只按 id 取人，新增不破坏）。

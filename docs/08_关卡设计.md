# 08 关卡设计文档（算法化生成 · 消除主线）

> 给关卡设计 / 算法团队。定义关卡的生成规则、难度曲线、节奏（**设计视角**）。
> **核心特点：关卡由算法生成,不靠人工手搓。** 关卡设计师定义「规则和目标」,算法负责「生成符合规则、难度可控的具体关卡」。
> 本文是 `09_算法系统设计.md`（**算法实现视角**）的配套：08 讲「要什么样的关卡」,09 讲「怎么生成/求解/评估/调度」。
> 前置依据（**消除主线**,见 `04`/`06`）：交换相邻两格的 3 连消除、素材库 20+ / 同屏 4-6、4连/T-L/5连特效、受限障碍（冰/锁/果酱/果冻底层目标）、战场切割、**补充随机（无确定序列）**、步数上限。
> 设计原则对照 `01_设计哲学.md`,算法接口见 `05_技术架构.md`。

---

## 一、关卡设计的核心思想

**人定规则,算法生成。** 关卡设计师不一关关手搓,而是定义：
- 关卡由什么构成（初始棋盘 + 障碍布局 + 目标 + 步数）。
- 难度由什么决定（可量化的难度指标）。
- 目标是什么（过关条件 / 计分规则）。

然后算法据此**无限生成**符合「low floor / high ceiling + 目标难度」的关卡,并**按玩家水平个性化推送**。

这是本项目的护城河——普通团队人工堆关卡、难度靠粗放 A/B;算法做到无限生成 + 千人千面。

---

## 二、关卡 = 什么（生成对象，对齐 `09` §1.4 / `05` schema）

一个消除关卡 = **初始棋盘 + 通关目标 + 步数上限 + 补充随机分布**,**不含「确定的出现序列」**。

| 构成 | 说明 |
|---|---|
| **初始棋盘** `init_board` | 普通水果布局（保证开局无现成消除）+ **障碍布局**（冰/锁/果酱/果冻）+ **战场切割**（WALL / 隔断划出的区域）。 |
| **水果种类组合** `species_set` | 从 20+ 素材库选 **4-6 种**（守同屏认知上限）;不同关卡用不同组合 → 多样性。 |
| **通关目标** `objectives` | 见第五节,可 MIXED 组合。 |
| **步数上限** `move_limit` | 核心约束。绝大多数目标都在「M 步内达成」。 |
| **补充随机分布** `refill_distribution` | 顶部补充新水果的**随机权重**;关卡只规定"各种水果以什么概率补",**不规定"下一个出什么"**。 |

> ⚠️ **关键纪律（与合成 Core 的根本区别）**：消除是**带随机补充的序列决策**,关卡**没有确定的"水果出现序列"**——补充是实时随机的（见 `09` §0/§1.4）。因此「难度/最优解」都是**期望意义**下的（多次模拟取分布）,不是单一确定值。任何把关卡建模成「一段确定序列」的设计都是错的（那是合成 Core 的模型,见 `03`）——会让算法（MCTS/投票）全盘失效。

## 三、算法必须保证的三件事（对接 `05`）

对每个生成的关卡,算法引擎必须能：

1. **验证（期望）有解**（`validateSolvable`）：随机补充下,**至少有代表性玩家能以不低的概率通关** → **保 low floor**。开局即死局、目标物理上凑不够的关卡不能推（见 `09` 4.2 / FI2Pop 硬约束）。
2. **算出（期望）最优解**（`computeOptimal`）：高水平求解器（MCTS）的**期望最优表现**（最高期望分 / 最优特效凑法）→ **定义 high ceiling**,也是同题竞技比对基准。
3. **量化难度**（`scoreDifficulty`）：用「求解器组投票」给难度档（EASY/MEDIUM/HARD/EXPERT）→ 个性化推送、难度曲线（见 `09` 4.3）。

---

## 四、难度由什么决定（可量化指标，消除版）

关卡难度 = 这些因素的加权（由求解器组投票统一量化,见 `09` §4）：

- **水果种类数量**：同屏 4 种比 6 种简单（可凑对的目标更密）。
- **障碍密度与种类**：冰块层数、锁/果酱数量、果冻覆盖面积越多越难（`09` 2.1.1 用**密度梯度**做"前松后紧"）。
- **战场切割程度**：区域越多、通道越窄越难（须保证连通、无孤立死区,`09` 2.5）。
- **目标类型与量**：得分 < 收集 < 清障 < MIXED;目标量越大越难。
- **步数松紧**：步数相对目标越紧越难（消除的核心难度旋钮）。
- **特效可凑度**：盘面是否易凑 4连/T-L/5连（易凑则地板友好）。
- **地板-天花板差距**：贪心勉强能过、MCTS 远超 = 高天花板（`09` 4.4）。
- **局内节奏（解空间曲线）**：每步「能产生消除的有效交换数」的变化（前松后紧）,见第七节 / `09` 3.6。

> 难度不是单一维度,是这些的加权;算法据此「定向生成某难度档的关卡」。

## 五、关卡目标 / 计分（5-7 种，对齐 `04` 2.6 / `09` 1.3）

关卡可用不同目标类型（增加多样性,大多可计算）：
- **得分**（SCORE）/ **收集数量**（COLLECT,消 N 个某水果）/ **清障碍**（CLEAR_BLOCKER,清 N 个冰块等）/ **底层目标**（BOTTOM,果冻清层）/ **限定步数**（LIMITED_MOVES,几乎所有关都带）/ **掉落收集**（运物到底,谨慎）/ **混合**（MIXED）。
- **滚动 / 挖矿形态**：清掉可见区 → 下方滚上来,持续挖,收集够 N 个算赢（障碍更克制,见 `04` 2.7）。

**计分原则（对接 `04`/`02`）**：
- **连锁（cascade）与特效给指数级而非线性奖励**——鼓励"凑大连锁/特效组合",制造 high ceiling。
- 特效组合（直线×爆炸、彩球×彩球等）显著高分（奖励规划）。
- 过关只需达到「基础目标」（low floor,谁都能过）;高分靠「凑特效 + 打长连锁」（high ceiling）。

## 六、Low Floor / High Ceiling 在关卡里的体现

| | 关卡里怎么体现 |
|---|---|
| Low Floor | 每关「（期望）有解」,基础目标低（随便消也能过）;3 连人人秒懂;无角色/铭文也能玩（裸 Core,`09` §0）。 |
| High Ceiling | 每关有「远高于基础目标的期望最优解」（凑特效 + 长连锁）;高手逼近最优拿高分;同题竞技比谁更接近最优。 |

**关键**：同一关,新手「过得去」（地板）、高手「玩出数倍分」（天花板）。这个跨度由 `09` 求解器组投票保证（`lfhc_gap`,4.4）。

---

## 七、局内节奏：解空间曲线（微观节奏，对齐 `09` 3.6 / 5.1）

一关之内的「节奏」由**解空间曲线**刻画：每一步盘面上「**能产生消除的有效交换数**」随推进的变化。前期值大（开阔、随便消）→ 后期值小（逼仄、要精算）= **前松后紧**。求解器试玩时逐步记录该曲线,生成时把「曲线符合设计意图」作为软目标之一筛选（`09` 4.5）。

> ⚠️ 度量用「**能产生消除的有效交换数**」,不是「所有合法交换」,否则高估解空间、曲线失真（`09` 3.2 / 纪律 7）。

## 八、难度曲线与个性化（核心）

### 8.1 整体难度曲线（宏观节奏，`09` 5.3）
- **开局极平缓**（少种类、少障碍、步数宽松,绝不劝退）→ **逐步引入**（加种类/障碍/战场切割,收紧步数,加 MIXED）→ **始终守地板**（再难也「（期望）有解、能过」,难的是拿高分）→ **张弛有度**（按难度起伏曲线排,避免单调递增）。

### 8.2 个性化难度（算法的核心价值）★
- **给累的人 / 新手**：更友好的关（同水果密、空间宽松、步数足）→ 轻松爽（守 low floor + 心流）。
- **给高手**：有挑战的关（要精心凑特效/规划才接近最优）→ 满足心流和钻研欲（high ceiling）。
- **目标**：让每个玩家都待在自己的「心流窄缝」——不无聊、不挫败。这是「难度匹配」需求的实现,也是 Block Blast 等没重点做的差异化。

### 8.3 个性化的输入（基于行为，非性别，`09` 5.4）
近期表现（分数、最优解达成度、是否频繁卡关/重试）、规划深度（行为接近哪个求解器画像：贪心 / Beam / MCTS）。也可直接给玩家「难度选择」（休闲 / 标准 / 挑战）+ 行为隐式微调。

## 九、关卡节奏与多样性

- **水果种类轮换**：不同关卡从 20+ 库抽不同的 4-6 种组合 → 视觉与策略都有新鲜感,防腻。
- **目标轮换**：得分关 / 收集关 / 清障关 / MIXED / 滚动关交替,避免单调。
- **难度起伏**：紧张关与放松关交替（张弛有度,别一路高压）。
- **高光关当「高潮」**：偶尔出「高特效密度 / 长连锁潜力 / EXPERT 难度」的关,作为高手炫技、普通玩家挑战的亮点。

## 十、同题竞技的关卡（对接 Meta 排行榜）

- 竞技模式：所有人玩**同一个关卡、同一 `FIXED_SEED`**(随机补充序列对所有人一致),比谁分高 / 谁更接近最优解。
- **公平性(随机补充下,采用解法A=近似公平,行业标准)**:
  - ✅ 固定 seed 保证「随机源相同」(补充序列一致,没有谁被特别针对)。
  - ⚠️ **不保证「过程完全相同」**:补充序列按操作消耗,玩家操作不同 → 盘面逐渐分岔 → 严格说非绝对公平。这是统计意义的近似公平,休闲消除竞技业界普遍采用(开心消消乐/Candy Crush),玩家接受。
  - 若未来要绝对公平(可选):竞技关用「固定初始盘面+少步数」(解法B)或「彻底确定化」(解法C)。首发用 A。详见 `09` 5.5。
- 算法保证：该关卡有明确的(期望)最优解(比对基准),且地板低(人人能玩)天花板高(高手拉开差距)。
- 这把「关卡的最优解空间」变成可比的竞技,服务 to be somebody。

## 十一、给关卡 / 算法团队的关键提醒

1. **人定规则,算法生成**——产出是「规则 + 难度指标 + 目标类型」,不是「一关关的具体布局」。
2. **关卡 = 棋盘 + 目标 + 步数 + 随机补充分布,绝不是"确定序列"**（第二节）——这是消除与合成的根本区别,把它建模成确定序列会让算法（MCTS/投票）全盘失效。
3. **每关必须「（期望）有解 + 高天花板」**（第六节）——地板低保大众,天花板高留高手,缺一不可。
4. **难度不靠"能不能过",靠"拿多少分 / 凑多少特效"**——再难的关也能过（守 low floor）,难的是逼近期望最优。
5. **无解关在生成时就排除;运行时无合法消除则自动洗牌(带约束)**——生成阶段保证有解(`09` FI2Pop 硬约束),运行时卡死靠洗牌兜底(对接 07/09),不靠"宣判死局"。
6. **障碍严格受限**：只用规则清晰、状态确定的少数障碍（冰/锁/果酱/果冻）;**绝不引入蔓延/增殖型**（巧克力/魔法搅拌器）,否则状态爆炸、求解失效（`04`/`09` 纪律 6）。
7. **张弛有度**：难度有起伏,放松关与紧张关交替。

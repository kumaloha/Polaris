## 1. 一路的分歧与纠错（被排除的路 = 宝贵的设计决策）

> 记录"我（Claude）主张 → 用户纠正 → 最终共识"。这些纠错界定了设计边界。

### 1.1 产品定位：我加戏，被收回
- 我主张：先定产品定位、元系统、core loop、变现。
- 纠正：格局收回到关卡本身。先把关卡做好。
- 共识：先做关卡设计这一件事。

### 1.2 "养成"：被否，但留存线要保留
- 我主张：宠物做成 RPG 养成——等级 / 技能树 / 升级 PetCast / 组队战力。
- 纠正：「不要做养成」。那套是 midcore，偏离 casual。
- 再澄清：宠物是关卡内临时存在的活元素（例：会偷懒、影响你消除的兔子），不是带进带出、要喂养升级的角色。
- 最终共识：① 关卡内的宠物 = 活元素（活的障碍/帮手）；② 但"萌宠养成"作为第二条留存线仍然要做（不是 RPG 战力养成，形态后续定）。两条留存线都要。

### 1.3 "章节"：被否
- 我主张：章节化机制编排（世界/章节为骨架，每章一个主打机制）。
- 纠正：「不要做章节」——不要世界地图/叙事/meta 那层外壳。
- 共识：可以有"关卡排布的内在节奏"，但没有章节系统。

### 1.4 CCS 节奏：我臆想了"教学带"，被实战体验纠正
- 我主张：每个机制走"教学→变奏→施压→精通"，用一段关卡铺一个机制。
- 纠正（用户玩到 ~50 关的真实体验）：CCS 并不这样。它很快，几乎每关都给新内容/新体验；教学是就地、一关之内、甚至隐性的，没有"教学专区"；几乎每关就是不同的障碍物组合。
- 共识：节奏要快、密；"新鲜"前期靠新机制亮相，长线靠"已有障碍物的错落重组"（机制有限，编排无限）。

### 1.5 障碍体系：三次推翻
- 第一版：沿用"九种 overlays" + 七角色分类学 → 被否：「不要用 overlay 这种概念，它是上一版糊弄的，会误导」。
- 第二版：我自创 8 种障碍体系（追求"完备正交"）→ 被否：「太复杂，要符合 gem 消除机制，先调研业内」。
- 第三版：调研业内（基础障碍其实就 2-3 种，深度在特殊棋子）→ 被否：「不要二手资料，自己充分理解 gem 的本质」。
- 共识：障碍要极简；不堆种类；从 gem 本质出发自己推，而非搬别人结论。（障碍最终形态待定，见第 5 章。）

### 1.6 焦点：我格局太小
- 我主张：一直在"玩法/难度"维度深挖（三元素 → 难度地理 → 水文）。
- 纠正：「格局太小」。地形、障碍、普通棋子、特殊技能，都要符合美学、要好看、有设计感、符合直觉、主题点缀。要一个能指导 1000 关的 formula。
- 共识：关卡设计是整体的（玩法 + 美学 + 直觉 + 主题），不是难度优化。玩家是完形地体验一关的。→ 催生第 6 章 Formula。

### 1.7 工作方式约定
- 这是头脑风暴：先深度理解，再设计；不写代码、不跑代码。
- 用直接文字对话，不要 AskUserQuestion 卡片。
- 重实证与自己的第一性原理理解，轻二手资料。

## 2. 关卡设计的本质（核心）

### 2.1 设计感 = 每关是"一道题"，不是"一个数字"
- 坏关卡：同一个模板换数字（"消 30 个红的 / 给 25 步"）。玩家每关做的事一样，第 3 关就看穿。
- 好关卡：每关一个独特局面，要玩家理解 + 应对。
- 可验证判据：一关有设计感 ⟺ 有一个无法靠"无脑乱消"绕过的题眼；抽掉它，难度明显塌掉。
- 三条禁令：① 禁"消除 X 个某色"这类不制造局面的目标；② 禁纯消除无障碍关（教学也用障碍教）；③ 禁换皮重复关。

### 2.2 gem 三元素的本质：一条"控制力"光谱（随机 → 控制）
三种基本元素不是并列，是特异性 / 控制力递增的光谱：
- 普通宝石 = 可互换的纯颜色载体，无个性 → 随机性之源（棋盘强加给你、不能选的现实）。
- 特殊宝石 = 被赋予功能的宝石，由玩家用技巧从普通宝石"炼"出来 → "随机 → 控制"炼金的产物（线消=精准、炸弹=区域、颜色炸弹=颜色、水晶球=更大）。
- 特殊技能 = 棋盘之外的独立力量，完全特异 → 确定性之源（凌驾棋盘随机的玩家意志）。

```
普通宝石 ——————→ 特殊宝石 ——————→ 特殊技能
随机/被动           桥梁(技巧)           确定/主动
棋盘摆布我       我把随机炼成控制       我凌驾棋盘
```

- match-3 好玩的本质 = "随机"与"控制"两极的拉锯，特殊宝石是中间的桥。
- 两极都不好玩：纯随机=老虎机（无技巧）；纯确定=解谜（无惊喜/无爽感）。
- 好玩在中间那个过程——在随机里用技巧创造控制、最后引爆，把混沌驯化成秩序。

### 2.3 关卡设计 = 给"驯化随机"塑形
- 一关本质是一道 "随机 → 控制转化"挑战："在这些约束下，你能否把足够的随机（普通宝石），转化成足够的、特定形态的控制（特殊宝石/技能），达成目标？"
- 所有旋钮都是在调三元素张力：颜色数（随机度）、障碍（限制控制施展）、目标（需要哪种形态的控制）、步数（多少次转化机会）、技能配额（多少确定性保底）。
- ⇒ 障碍不是设计主体，只是"约束控制力"的一个旋钮。这是之前堆障碍堆错的根因。
- 设计感 = 每关的转化挑战长着不一样的形状（精准控制 / 大规模控制 / 压力下快速建立控制…）。

### 2.4 难度地理 / 补给水文（地形 + 障碍 + 棋子分布）★ 关键突破
> 由用户的"沙漏关"例子点醒。这是比 2.3 更落地的设计抓手。

- 旧焦点（宝石排布）错在只看静态第 0 帧。gem 有重力——真正决定玩法的不是初始那帧，而是消除之后补给怎么流动。
- 把"地形 + 障碍 + 棋子分布"统一成 重力场里的"补给水文学"：
  - 地形 = 河道：画出补给往哪流。宽口=上游（补给足、易制造消除）；窄口=咽喉（流量瓶颈）；窄口之下=下游/死水（补给到不了）。
  - 障碍 = 堰坝 / 礁石：既改变补给流动（如棉花糖把一块区域堵成死水），本身又常是目标。障碍的难 = 它本身硬度 × 它在水文图上的位置。
  - 棋子分布 = 初始水位 / 下手点：哪里是能动的普通宝石，决定你从哪开始往棋盘注入控制。
- 一关的设计 = 画一张水文图，把目标钉在你要它难的水文位置上。
- 死水区 = 控制力最难到达的地方（接回 2.3）。

贯穿案例 · 沙漏关：地形上宽、中腰窄、下展开；蜂蜜（目标）+ 棉花糖（阻挡）全堆下方。因为重力，补给要穿过中腰窄口才能喂到下方，棉花糖又把下方堵成死水 → 难点天然钉在下游。设计师用 地形 + 障碍位置 + 棋子分布，把难点钉死在补给下游。

### 2.5 完形体验：一关的骨架是一个"四位一体的形状"★ 最高本质
> 由"格局太小"的批评催生。这是统一玩法与美学的终极洞察。

- 玩家整体地、一瞬间形成印象（好看 / 我大概懂 / 有意思 / 有味道），这是一个完形（gestalt），不是四张分开的评分表。割裂地只优化难度必然格局小，因为玩家从不割裂体验。
- 一关真正的骨架是一个形状（地形+障碍+分布的轮廓）。设计感 = 这个形状同时是四样东西：
  1. 难度地理（玩法）——形状决定水文、难点位置。
  2. 视觉构图（美）——形状本身平衡、有节奏、好看。
  3. 直觉指引（可读）——形状一眼告诉玩家目标在哪、难点在哪、往哪使劲。
  4. 主题表达（风味）——形状是某主题母题的化身。
- 多维共振于同一个形状 = 设计感的本质（不是四维各自达标，而是一个形状四件事全干）。
- 沙漏关验证：下游死水（玩法✅）+ 对称优美（美✅）+ 窄腰自己喊"下面难"（直觉✅）+ 沙漏=时间/沙可挂主题（主题✅）。一个形状，四件事全干。

## 3. 方法论工具箱

### 3.1 设计感三杠杆
- 局面（静态）：棋盘形状 + 障碍布局 + 目标位置，开局即出题。← 静态障碍的物料。
- 活变量（动态）：会动/会变/会捣乱的元素，逼你边消边应对。← 动态障碍/活元素（宠物）的物料。
- 咬合（涌现）：元素互相作用产生连锁与破局点，玩家发现解法。← 特殊宝石组合。
- 高级关卡 = 三杠杆叠加。

### 3.2 双难度 + 相对化系数
- 难度是两维，分开调：认知难度（要想多久看懂怎么破）vs 执行难度（看懂了手上做不做得到）。平庸只会抠步数（执行），高级在认知上做文章。→ 设计感与难度本是一回事：让关卡更难的正路是"更需要想"。
- 所有参数相对化，绝不设绝对值（反数字膨胀）：
  - 步数 = 理论最小步数 × 宽裕系数 S
  - 分数 = 每步密度 D × 步数
  - 障碍 = 覆盖率 O + 种类数 K（K 极少 > 3）
  - 认知难度 ← O / K / 形状复杂度；执行难度 ← S / D。

### 3.3 节奏
- 锯齿波 + 前松后紧：虐-松-虐-松，整体爬升，新机制引入时回弹放松；几个标志性"墙"做成就记忆点（学公平的墙，不学逼氪的墙）。
- 节奏密度 = 内容 ÷ 关卡预算：CCS 是为无限长跑而"稀释"（一个机制统治几十关 + 大量换皮）；我们靠"生成系统的多样性"在规模上维持密度，不靠稀释。
- 错落有致（障碍布局的五种错落）：空间错落（疏密聚散、留操作区）、层次错落（血量/层数差）、种类错落（主次而非平均堆）、功能错落（静+动搭配）、关间错落（复杂度起伏）。

## 4. 两个本质约束（之前漏掉、必须扎根）

### 4.1 match-3 本质：随机 + 爽感 + 轻策略（别做成硬核解谜）
- 随机性是灵魂不是 bug：题无唯一解；随机性还是留存缓冲（卡关时"再试一次也许运气好"给了继续的理由）。
- 爽感（juice）：casual 玩家图的是爽——消除反馈、连锁 cascade、特殊宝石大爆发、绝杀清场。全程烧脑没爽点的关卡，哪怕"有设计感"也是失败品。
- 设计感三杠杆是"认知侧"，须再配爽感：布局要引导玩家走向连锁高潮、攒大招、收尾大爆发。判据加一条：通关全程有爽感节奏。

### 4.2 hybrid 本质：关卡设计要带留存
- hybrid = LTV = 留存 × 变现；关卡最终为留存服务，不只是"好玩"。
- "模拟运营人工生成"的真意：生成系统是 live ops 的内容引擎——持续供给新关卡、活动关，按数据动态调难度。不是一次性生成静态序列。
- 难度墙不只是成就记忆点，也是变现/回访点；难度曲线本就是一条变现曲线。
- 我们的留存策略：让"关卡系统本身"成为留存引擎（持续供给有设计感、有爽感的关卡 + 运营调控），第二条腿是萌宠养成。

## 5. 障碍物：业内真相与我们的态度

> 注：此节是过程性结论。障碍最终形态待定——共识是"从 gem 本质自己推、保持极简"，不照搬下表。

- 业内真相：CCS / Royal Match 看着障碍一大堆，机制其实就几个，剩下全是皮肤和参数变体。
  - 静态三类（覆盖型 / 箱子型 / 锁包型）本质同源——都是"占位、需用消除处理掉的东西"，区别只是处理方式（消其上 / 消相邻 / 消包含）。可收敛到 1-2 个机制。
  - 动态类少而精：扩散 / 生成 / 倒计时 / 运输。
  - Royal Match 口碑更好不是因为障碍多，而是把力气全压在特殊棋子（火箭/炸弹/灯）的两两组合上。
- gem 硬约束：好障碍绝不引入新操作，只给玩家本来就在做的"交换+消除"动作附加一层意义。（传送带、运送出口等在往机制上硬加规则，所以"复杂"。）
- 重心结论：障碍极简，真正的深度押在「特殊宝石的生成 + 两两组合」+「地形/布局」上。

## 6. THE FORMULA（可指导 1000 关）

### ① 核心公式（什么是一关）
> 一关 = 一个"四位一体的形状" + 在其上的编排
> 形状 = 难度地理(玩法) × 视觉构图(美) × 直觉指引(可读) × 主题表达(风味)
> —— 四者必须在同一个形状上对齐共振，而非各自达标。

### ② 生成公式（怎么造一关）
自上而下，每一步都过 【玩法 ∧ 美 ∧ 直觉 ∧ 主题】四重对齐：
```
主题母题      给形状以风味和由头（沙漠/冰窟/糖厂…）
   ↓
地形原型      选一个四位一体的形状（沙漏/瓶颈/分叉/孤岛/断层…）
   ↓
障碍编排      在形状上钉难点 + 视觉节奏 + 主题皮肤
   ↓
棋子分布      定切入点 + 色彩构图
   ↓
控制工具      特殊宝石/技能 = 安全阀 + 爽点
```
- "符合直觉"是美学与玩法的接缝：要求 视觉焦点 = 玩法焦点——视觉上最突出的地方，必须就是玩法上最关键的地方。错位→玩家困惑；对齐→关卡"自己会说话"。

### ③ 规模公式（怎么撑 1000 关不换皮）
> 主题组 × 地形原型库 × 障碍组合 × 分布变体 = 组合爆炸
> 主题成组推进，地形原型组内复用变奏，难度沿锯齿曲线螺旋。
> 咬合项目架构（加机制 = 新文件 + 注册行）：每加一种障碍/主题，组合空间翻倍 → 规模不靠稀释、靠乘法。

### Formula 的格局
地形、障碍、普通棋子、特殊技能，每一个都不再只为难度服务，而是同时为难度、美、直觉、主题服务。设计一关 = 雕一个四位一体的形状，再为它挑主题、上皮肤、配色彩。

## 8. 速查：判断与禁令

- ✅ 每关是一道题，有抽掉就塌的"题眼"。
- ✅ 视觉焦点 = 玩法焦点（直觉接缝）。
- ✅ 一个形状四位一体（难度 × 美 × 直觉 × 主题）。

# 02 META 系统（可复用框架）

> 本文档定义 Meta 元系统。**Meta 独立于具体 Core 玩法,是可复用框架**——同一套 Meta 未来可驱动合成、消除等不同 Core。
> Meta 与 Core 的连接只通过「资源流」：Core 产出分数/资源 → Meta；Meta 提供能力/加成 → Core。技术接口见 05。
>
> 设计依据请对照 `01_设计哲学.md`。本文档只写「是什么、怎么做」。

---

## Meta 总览

> **主题与角色解耦**:消除元素是水果;角色是独立的二次元人物,承载技能+铭文+收集+抽卡。风格偏二次元(角色立绘 + 水果萌系,见 `06`)。

Meta 以**角色**为核心,由这些子系统组成:

| 子系统 | 类型 | 作用 | 对应玩家需求 |
|---|---|---|---|
| 角色系统(二次元人物) | 质变·主动一次 | 一个技能改变玩法,一次带一个 | 难度匹配 / high ceiling |
| 铭文(挂在角色上) | 量变·被动持续 | 一套铭文绑一个角色,可配置搭配 | 确定性成长(角色养成) |
| 抽卡系统 | 获取 | 靠高分解锁,获得角色(首抽必出借贷) | 确定性成长(实力驱动) |
| 皮肤系统 | 外观 | 纯审美,变现 | To be somebody |
| 排行榜/竞技 | 社交 | 同题竞技,被看见 | To be somebody |

**核心循环**:玩消除(水果)→ 打高分 → 高分给抽卡机会 + 铭文碎片 → 抽角色(首抽借贷,后续随机)+ 养所带角色的铭文 → 用角色技能(一局一次,第二次看广告)+ 该角色铭文加成 → 玩更高分 → 上周赛被看见。

---

## 一、角色系统（核心 · 二次元人物）★

> **主题与角色解耦**:消除元素是**水果**(见 `04`/`06`),角色是**独立的二次元人物**(不是拟人水果)。角色承载技能 + 收集 + 抽卡 + 铭文养成。这是品类标准做法(如 Candy Crush 的糖果与 Tiffi 是两层),且解决了"水果难角色化"的错配。

**定位**:每个角色 = 一个二次元人物,带一个「改变玩法」的主动技能 + 一套可配置的铭文(见二)。

**三条铁律**:
1. **一局只能上一个角色**(强制流派选择)。
2. **角色技能一局只用一次**(强力但不填平难度——一次性的强,不是持续开挂;是「这一次王牌留到何时用」的临场决策)。
3. **第一次免费,第二次起看广告续用,且有上限(1-2 次)**——超过上限不能再续(否则破坏「一局一次」这个难度基石)。这是「广告」变现腿的落点:卖机会(用时间换),非强度。

**生死线**:
- **锦上添花非雪中送炭**:没带任何角色,光靠裸 Core 也能正常玩、能过(守 low floor,也是 `09` 算法评估基准)。
- (消除掉落一次一大批、位置难算 → **不做「预知未来棋子」**,实测没用;**不单做清障流**——障碍由消除/特效处理。)

**4 个流派(首发,一次带一个)**:

| 流派 | 技能 | 说明 | 强度 |
|---|---|---|---|
| 高风险流 ★核心 | **借贷** | 见下,等价交换的化身 | 厉害 |
| 容错流 | **时间回退** | 回退最近 5 步 | 厉害 |
| 容错流 | **存档快照** | 存一个局面,可一键跳回 | 厉害 |
| 操作流 | **隔位对换** | 正常只能换相邻 1 步,这个能换相邻 2 步(隔一个) | 厉害 |
| 特效保护 | **彩球护盾** | 这局保彩球一次(第一次被碰只掉护盾,彩球保留) | 中 |
| 下落系 | **重力翻转** | 翻转棋盘重力一次,全部反向下落重排局面 | 厉害 |

> 注:回退与存档都做,但**作为两个角色,一局只带一个、各只用一次**——靠「带一个+用一次」双重约束,容错不泛滥(不违背不信廉价成长)。
> (爆炸放大已归**铭文**特效线,见二——它是被动量变,非主动一次。)

**借贷技能(核心,等价交换化身)**:
- **借**:立刻获得一个特效(4 连直线 / T·L 爆炸 / 5 连彩球 的效果)。
- **还**:**本关内必须还**——还的方式 = 让你某个特效失效,降格成普通棋子(**玩家自选**牺牲哪个,策略:还不重要的那个)。
- **罚**:**关末若有未还的债 → 不算过关**(硬约束,逼玩家规划好借与还)。
- 完整 LFHC:新手简单借个特效还得起;高手精算「借什么、何时借、还哪个最优」。强度受「还债」代价制约,好用但不离谱。

> 角色 = 二次元人物 + 能力载体,**非叙事载体**(不做剧情)。靠美术(可外包,二次元立绘)做辨识度 + 一句话能力标签。
> (候选技能池另有:斜向消、控制下落方向、跨区消除、抓取暂存、连引、抵押消除等,均按「一局用一次」设计,后续做 LiveOps 持续出新角色。)

## 二、铭文系统（量变层 · 挂在角色上、可配置）

**定位**:**一套铭文挂一个角色**(不是全局共享)。带哪个角色,就用这个角色身上配置好的铭文。换角色 = 换技能 + 换这套铭文配置。**养一个角色 = 练它的技能 + 调它的铭文搭配**(契合二次元手游的"角色 + 角色身上养成",如御魂/圣遗物)。

**分类标准(铁律)**:**被动持续 = 铭文;主动用一次 = 角色技能**(见一)。

**可配置 + 取舍**:每个角色的铭文有**有限点数**,玩家在这个角色身上**搭配着点几条线、放弃其他**——「搭配」本身就是取舍(不信廉价成长),也制约任何单条线。收集来的角色因此各有各的 build。

**获取**:**通关分越高 → 得到的铭文碎片越多**(与抽卡同逻辑,全靠实力/高分驱动)。碎片用于强化所带角色的铭文。
> 注(数值向):铭文绑角色后,养成成本随角色数增加——给了长期养成空间(强化收集),但碎片产出要够养,别太肝(原型/运营调)。

**4 条线**(数值原型调):

1. **步数线**(核心资源,走「奖励技巧」而非「白给」)
   - 主机制 = **连锁奖励步数,且递进式**:铭文等级越高,触发奖励所需的连锁数越低(1 级:连锁 8 才奖 1 步;高级:连锁 5 就奖 1 步)。
   - **始终要靠「打连锁」这个技巧才拿到步数**(不是开局白送)→ 奖励高水平,不填平难度。

2. **特效线**(接多连特效)
   - 特效得分 / 范围 / 连锁加成提升。
   - **爆炸放大**:让已有爆炸/彩球的作用范围变大(被动量变,故归铭文)。
   - 攒满:开局随机送 1 个特效,或特效组合威力额外升一档。

3. **产出 / 资源线**(接核心循环)
   - 分数 / 抽卡资源 / 铭文碎片产出提升。
   - 攒满:每日首通双倍资源,或高分通关额外抽卡机会。

4. **战场 / 空间线**(接障碍与战场切割)
   - 障碍血量 -1(冰块少 1 层)、跨区消除小优化、开局清除少量随机障碍。
   - 攒满:开局自动破 1 个障碍,或某隔断初始打开。

**搭配 × 角色技能 = 双层 build**:在某角色身上配什么铭文,和它的技能协同——如爆炸放大铭文 + 爆炸保护(彩球护盾)角色;步数线 + 借贷角色(多步数从容还债)。

**边界(防填平难度)**:加成小幅缓慢、攒满质变克制。判断标准(`01`):点了铭文是「更轻松地过同样的关」(可以,量变给余地)还是「难度被抹平随便过」(不行)。**步数线尤其守住「靠连锁技巧才给」,不直接白送步数。**
> (已砍「容错线」:无代价容错与「不信廉价成长」冲突,且容错由角色的回退/存档承担。)

## 三、抽卡系统（获取层）

**定位**：角色通过抽卡获得,**抽卡机会靠「把关卡分玩得很高」解锁**(实力驱动,非花钱/等时间)。

**首抽保底**：**初始抽卡 100% 出借贷**(如阴阳师首抽必出雪女)——所有新玩家从同一个好用的核心技能起步,新手体验可控,且自然形成「借贷为主力」(90% 玩家用借贷,10% 用抽到的特色技能)。

**后续随机**：首抽之后,抽卡为随机池(其他技能),靠高分解锁机会。

**与原则一致**:靠实力解锁的抽卡不违背「不信廉价成长」;借贷保底「人人能玩」(雪中送炭由它兜底),抽到的其他技能是「锦上添花」→ 抽卡不必需,玩家不焦虑、不逼氪。

**注意**:抽卡给「角色(玩法能力)」;外观(皮肤)是另一条线(见四)。能力靠抽卡,外观靠买,严格分开。

---

## 四、皮肤系统（外观/变现层）

**定位**：纯外观,不带任何数值/能力加成。变现的干净第三条腿。

**设计原则**：
- **绝不带数值/能力加成（铁律）**：一旦沾强度,就破坏公平、违背原则。皮肤管外观,角色管能力,严格分开。
- 「换皮肤仍可识别」：物种（类别）+ 进化阶段（等级）靠核心轮廓识别,皮肤只换表面装饰,不改可识别结构。一只猫无论什么皮肤,仍一眼认出是猫、是成体。
- 无限可做、零平衡负担：节日皮、联动皮、稀有皮,持续出,只需美术（可外包）。

**作用**：
- 喂「to be somebody」：外显身份和炫耀（高手顶稀有皮肤的动物上排行榜）。
- 长期变现 + 持续新鲜感来源。

## 五、排行榜 / 竞技系统

**定位**：承接 high ceiling,让「玩得好」被看见。

**设计**：
- **同题竞技**：同一个关卡/同一局面,所有人比谁的解更优（更高分/更少步/更高级组合）。把 Core 的「最优解空间」变成可比的竞技。
- 这天然契合算法（同一道题比谁解得好,本质是把竞赛搬进休闲游戏）。
- 前提：Core 必须有「客观优劣」（分数/步数可比）——合成、消除都满足。
- 配套：完美通关标记、好友分数对比、稀有皮肤展示——让高手的炫技有舞台。

**为什么重要**：To be somebody 是三大需求之一,而多数休闲游戏这块做得弱——这是本项目可强化的差异点。

## 六、Meta 与 Core 的连接（接口概览）

**只通过资源流连接,不依赖 Core 的具体玩法：**
- Core → Meta：Core 产出标准化的「分数」和「资源」。
- Meta → Core：Meta 提供「角色能力（一个）」和「铭文加成（一组数值）」给 Core 生效。

**这意味着**：Meta 不关心 Core 是合成还是消除,只关心「收到多少分/资源」和「要把哪个角色能力、哪些铭文加成注入当前局」。技术实现见 `05_技术架构.md`。

**防割裂**：Meta 与 Core 机制可以无关（如 Gossip Harbor 的合成与庄园）,但要靠「一致的动物主题包装 + 顺畅的资源流」让玩家感觉是一个游戏。

## 七、变现总览

三条腿,全部不卖强度：

| 变现 | 对象 | 卖什么 | 原则 |
|---|---|---|---|
| 广告 | 大众 | **续用角色技能的机会**(第二次起,有上限)+ 激励/插屏 | 卖机会(用时间换)非强度;玩家主动想要时看,不强制 |
| 抽卡 | 想要能力的人 | 角色（玩法能力）| 靠高分解锁,实力驱动;首抽必出借贷 |
| 皮肤 | 想炫耀的人 | 纯外观 | 绝不带强度,喂 to be somebody |

**红线**：任何变现都不卖数值强度、不破坏公平、不违背「不信廉价成长」。三腿全是「卖机会 / 卖外观」,无一卖永久强度。

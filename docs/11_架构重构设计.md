## 0. 设计目标一句话

把 level.gd 从"所有新需求都往里塞的 3533 行"收敛为"~800 行编排者 + 三个注册表扩展点"，使**加一种障碍 / 加一只宠物 = 新增一个文件 + 注册一行**，主干文件从此不再增长。

三条铁律（所有契约共同遵守）：

1. **状态闸门集中**：`_busy / _settled / generation` 只住在 level.gd，唯一写锁点；子模块经信号请求、只读不写。（review C1/C2 竞态的制度性根治）
2. **生命周期对齐场景树**：凡持有 tween/动画状态的模块必须是 level 的子 Node——换关 `free` 即自动断 tween，跨代回调在结构上不可能发生。纯计算模块用 `static func`（level_motion.gd 已是范例）。
3. **单向依赖**：`app → {match3, meta}`；`match3 ⊥ meta`（对局永远不知道养成存在）；所有视图模块 ⊥ 彼此（只消费 StepReport，不互相调用）。

## 2. 契约 A：StepReport —— 级联步数据流脊柱

### 2.1 现状（引擎已经吐什么，不发明新数据）

引擎两个函数已经产出全部所需数据，本契约只是**正式化命名 + 固定分发顺序**：

- `ME.collect_clears()`（match_engine.gd:1196）返回：
  `{ to_clear: Array[Vector2i], spawns: Array[{pos,kind}], triggered_spawns: Dict, triggered_spawn_fx: Dict }`
- `ME.account_clears()`（match_engine.gd:1304）返回：
  `{ by_species: Dict, jelly_cleared, blocker_cleared, choco_cleared, bomb_defused, popcorn_hit, cake_destroyed, mystery_revealed: int, locked: Array, cake_blast: Array, ... }`
  （计数器全集即 board.gd:200 的 `SNAPSHOT_COUNTERS` 9 项）

现 `_resolve_cascades`（level.gd:2873-2945）在循环体内**内联**完成组装+消费全部职责；重构后它只组装 report 并按序分发。

### 2.2 StepReport 字段定义

保持引擎的 Dictionary 风格（不引 class，两端镜像文化如此）：

```gdscript
# StepReport: 每个级联步的只读数据包。level.gd 组装，按固定顺序分发。
{
  "cascade_level": int,            # 本级联深度，1 起
  "to_clear": Array[Vector2i],     # 实际消除格（已剔 locked、已并 cake_blast）
  "spawns": Array,                 # [{pos: Vector2i, kind: int}] 本级生成的特效棋子
  "protected_spawns": Dictionary,  # set: spawn 中保留节点只叠 shine 的格
  "triggered_spawn_fx": Dictionary,# pos → kind: 同步触发特效的方向覆盖
  "account": Dictionary,           # account_clears 原样返回值（含 9 计数器/locked/cake_blast）
  "score_gained": int,             # 本级 score_for_clear 结果
}
```

### 2.3 分发时序（固定顺序，写死在 level.gd）

```
while 有匹配:
    board 推进（collect→account→accumulate→_apply_clears，原顺序不变，level.gd:2902-2929）
    report = 组装 StepReport
    overlays.on_step(report)        # 同步：果冻碎/冰裂/炸弹拆/爆米花受击 等障碍演出
    hud.on_step(report)             # 同步：目标进度/分数/星级
    skills.on_step(report)          # 同步：by_species 充能（现 _charge_skills，level.gd:2911）
    await board_view.play_step(report)   # 异步：消除动画+spawn shine+下落补充（唯一 await 点）
```

规则：
- 消费者**只读** report，不回写、不互相调用；
- 只有 board_view 是 await 点（它是演出节拍）；overlays 的受击演出自己起 tween 但不阻塞主循环；
- account_clears 原地递减障碍层数组的副作用**保持现状**（与 board.try_swap 同口径，level.gd:2902 注释的既有契约），overlays 读到的层值天然是"已扣减后"的——所以 overlay 演出依据是 `report.account` 里的计数差量 + 自查 `current_value()`，见契约 B。

### 2.4 迁移说明

纯搬运重排，行为零变化：`_resolve_cascades` 循环体内的 `_refresh_jelly_visuals/_refresh_coat_visuals`（level.gd:2909-2910）就是 overlays.on_step 的雏形；`_charge_skills` 就是 skills.on_step；节点同步段（level.gd:2930-2944）归 board_view.play_step。

## 4. 契约 C：PetCast 基类 —— 宠物施法扩展点

### 4.1 时兔现状生命周期（源码实测，level.gd:1582-2022）

```
_on_skill_pressed(idx)                          # level.gd:1582，按 SKILLS[idx].skill 字符串 dispatch
→ _skill_time_rewind()                          # :2021 仅设 _time_rewind_cast_pending=true（不锁 _busy ← C1）
→ _play_time_rewind_pet_animation()             # :1650 建 rig(帧精灵+沙漏) 挂 skill_bar
→ rig.set_meta("cast_tween", t)                 # tween 链：出框帧序列→跳跃12段→施法帧→HOLD
→ tween_callback(_commit_time_rewind_cast)      # :1890± board.skill_rewind() + _render_board(false)
→ 归位序列 → tween_callback(_retire_time_rabbit_rig)  # :1994 回收 rig、复原头像
```

已知缺陷（review C1/C2）：施法不锁 `_busy`；`load_level/_exit_tree` 只杀 opening tween 不杀 cast tween；callback 无 generation 守卫。
解剖结论：33 个函数 + 51 个常量中，**机制（tween 驱动/锚点解析/rig 回收/生命周期）约占 30%，每只宠物相同；数据（帧序列/锚点/时长）约占 70%，每只宠物不同但形状一致**。

### 4.2 基类设计

```gdscript
# match3/pets/pet_cast.gd
class_name PetCast
extends Node2D
## 宠物施法控制器基类。实例由 skills.gd 创建并挂在 level 之下：
## 换关 free(level 子树) → _exit_tree → cancel() → tween 自动死 → 跨代回调结构性不可能。

signal cast_started      # → level: 上输入锁（唯一写锁点在 level 侧）
signal cast_committed    # 技能效果已落地 → level: 同步棋盘视图
signal cast_finished     # 演出全部结束(含归位/取消) → level: 释放锁

enum State { IDLE, CASTING, COMMITTED, RETIRED }
var _state := State.IDLE
var _tween: Tween                      # 绑定在 self 上：create_tween() 即可，self free 即死

## ── 模板方法（基类实现，子类勿覆写）──
func start_cast() -> bool:
    # IDLE 检查 → _state=CASTING → emit cast_started
    # → rig=_build_rig() → 按 _cast_script() 编排 _tween
    # → 中点 tween_callback(_commit) → 末尾 tween_callback(_finish)
func cancel() -> void:
    # 任何状态可调：杀 tween → 立即回收 rig → _restore_avatar()
    # → 若未发过 finished 则 emit cast_finished → _state=RETIRED
func _exit_tree() -> void:
    cancel()   # 换关兜底，结构性根治 review C2

func _commit() -> void:
    if _apply_effect():
        emit_signal("cast_committed")
    _state = State.COMMITTED

## ── 子类钩子 ──
func _build_rig() -> Node2D            # 组装演出节点树（精灵/道具/特效）
func _cast_script() -> Array           # 声明式施法脚本，见 4.4
func _apply_effect() -> bool           # 技能落地：调 board.skill_*，返回是否生效
func _restore_avatar() -> void         # 复原头像位显示（取消/结束共用）
```

### 4.3 锁协议时序（与 level.gd 的完整交互）

```
玩家点头像
  → skills.gd: PetRegistry 查类 → 实例化挂 level → pet.start_cast()
  → pet: emit cast_started      → level: _busy = true          ← 修 C1（结构化）
  → pet: tween 演出...
  → pet: _apply_effect()=board.skill_rewind() → emit cast_committed
                                 → level: board_view.rebuild()  (或增量同步)
  → pet: 归位演出... → emit cast_finished
                                 → level: _busy = false
换关/重试:
  level.load_level → 子树 free → PetCast._exit_tree → cancel()
  tween 随节点死，不存在残留回调；锁由 load_level 统一重置。   ← 修 C2（结构化）
```

generation 守卫**不再需要**：tween 绑在 PetCast 自身（`create_tween()` 默认绑定 self），节点死 = tween 死 = 回调死。这是"用对象生命周期替代手工代际检查"的根治。

### 4.4 施法脚本数据格式（51 个时兔常量的归宿）

```gdscript
# match3/pets/time_rabbit.gd（示意）
const CAST_SCRIPT := [
    # 命名锚点由基类解析：home(头像槽) / frame_lip(框沿) / cast(施法位) / board_center
    {"tex": K1_PEEKTOP, "anchor": "frame_lip", "offset": Vector2(0, -8), "dur": 0.10, "ease": Tween.EASE_OUT},
    {"tex": K2_PEEK,    "anchor": "frame_lip", "offset": Vector2(0, -34), "dur": 0.12},
    # ... 跳跃段可声明为 {"jump_to": "cast", "points": 12, "arc": -118.0, "dur_each": 0.06}
]
```

**克制声明**：这就是字典数组，不是动画 DSL。命名锚点 + 贴图 + 时长 + 缓动四要素覆盖时兔现有全部演出（实测其 `_time_rabbit_jump_points/_durations` 本来就是数据表的展开写法，level.gd:1796-1853）。出现第二类完全不同形态的动画前，**禁止**给脚本格式加能力。

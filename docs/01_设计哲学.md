# 01 设计哲学

> 本文档解释「为什么这么设计」。每条都是一个设计决策 + 它的依据 + 要避开的陷阱。
> 做任何新设计决策时,回到这里对照,确保不偏离。

---

## 一、为什么是 Low Floor / High Ceiling

**决策**：玩法表面极简(大众能玩),底层有深度(高手能钻)。

**依据**：
- 大众盘要「上手无门槛、能放松地玩」;但纯简单留不住人、没深度。
- 高手/钻研者要「有东西可优化、可炫耀」;但纯硬核吓跑大众。
- LFHC 用一个游戏覆盖两端,不必二选一。Block Blast（全球下载第一）的官方定位就是 "accessibility + strategic depth"——已被验证可行。

**陷阱**：
- 把深度做在「表面难度」上 → 地板升高,大众流失。**深度要藏在 high ceiling（给想要的人）,不能顶在所有人脸上。**
- 把「优化/最优解」当成通关门槛 → 不会优化的人被卡死。**最优解是「锦上添花的上限」,不是「入场券」。**

## 二、为什么「不信廉价成长」是核心

**决策**：任何成长都要付出真实代价（互斥/抵押/牺牲）,拒绝无代价的数值养成。

**依据**：
- 这是创始人的真实信念,也是创作的诚实——只能做自己真信的东西。
- 现实里成长有代价,无代价的成长是假的,玩家（尤其经历过现实的人）不信、不珍惜。
- 有代价 = 有取舍 = 策略,而策略正是本项目的认知正中心。

**怎么落地**：成长 = 有代价的选择。例：
- 互斥：点了这条线永久关闭另一条。
- 抵押：现在变强,未来还债（如「借贷牌」：抵押一张换一张更高的,以后要还）。
- 牺牲：消耗已养成的东西换更强的。

**陷阱**：别把这个原则做成「惩罚玩家」。代价是「有意义的取舍」,不是「无谓的难受」。玩家要感到「我做了个艰难但值得的选择」,不是「这游戏老坑我」。

---

## 三、为什么 Meta 与 Core 解耦

**决策**：Meta（元系统）和 Core（小游戏玩法）在设计和技术上都分离。Meta 可复用,Core 可插拔。

**依据**：
- 经实证（Gossip Harbor 等）：成功的休闲游戏,core 玩法和 meta 常常机制无关,只靠「资源流」连接（玩 core 赚资源 → 投入 meta）。
- 解耦的红利：一套 Meta 框架做一次,可驱动多个 Core（合成、消除……）。这是「做多个游戏」的正确方式——复用 meta,换 core,而非每个从零做（也不是赌爆款式的盲目铺量）。
- 同时复用两个资产：算法引擎 + Meta 设计。

**陷阱**：
- 解耦≠割裂。两者要靠「资源流 + 一致的主题包装」让玩家感觉是「一个游戏」,不是「两个 App」。Gossip Harbor 的合成和庄园机制无关,但玩家不觉得割裂——学它怎么缝。
- 「框架可复用」是第一个产品验证成功后的红利,不是第一步的捷径。**先把第一个做成,框架才被证明。** 否则是新版的「赌量」幻觉。

## 四、为什么 Meta 是「策略构筑型养成」,不是「数值养成」

**决策**：Meta 用铭文/角色/build 这类「策略构筑」,不用等级/强化/装修/剧情这类「数值养成」。

**依据**：
- 养成分两派：① 数值养大派（等级、强化、收集、装修、剧情）—— 偏堆量/情感/陪伴 ② 策略构筑派（铭文、技能树、build、卡组）—— 偏脑力/规划/取舍。
- **它们都是养成**（都在「成长、积累、变强」,都能留人）,但创始人精通且热爱的是「策略构筑派」,不熟也不信「数值养大派」。
- 关键洞察：**铭文 = 策略类的养成**。它既是养成（有成长感、留人）,又是策略（有限资源最优组合、有取舍）,还天然有代价（选这个放弃那个）,且不需要叙事/笔力、不变重。完美契合所有要求。

**陷阱**：
- 不碰剧情/叙事。理由：① 创始人明确不做讲故事类型 ② 叙事吃笔力（团队短板）③ 加剧情会让游戏变重。**注意：剧情/角色故事确实是「玩家与游戏建立情感连接」的强力手段（原神、泡泡玛特同理）,但这是「未来有多个成功游戏后」才考虑的方向,现在不做。**
- 角色仍然做,但「角色 = 能力载体 + 视觉记忆点」,不是「叙事载体」。靠美术（可外包）做辨识度,不靠故事。

## 五、为什么质变放角色、量变放铭文

**决策**：
- **角色 = 质变 = 改变玩法**（预知未来牌、判死局、时间倒流、借贷牌等）。一次只能带一个。
- **铭文树 = 量变 = 放大**（加面积、加时间、加容错）。慢慢攒,攒满有轻微质变。

**依据**：
- 「改变玩法」的能力若可叠加（像铭文那样全点满）→ 各种能力齐发 → 玩法被外挂填平 → 难度崩。
- 把质变放角色 + 「一次只带一个」→ 强制玩家选流派（这局走「预知流」还是「倒流流」还是「借贷流」）→ 再强也只有一种能力 → 难度守住,且形成 build 选择。
- 量变（放大）可以叠加,因为它不改规则,只是放宽余地,不会消解难度。

**陷阱**：
- **角色能力必须「锦上添花」非「雪中送炭」（生死线）**：没带任何角色,光靠 core 也能正常玩、能过。角色提升上限,不构成门槛。否则抽卡的随机性会卡死没抽到角色的新玩家,破坏 low floor。
- **「判死局」类能力做成「预警/避免」,不做「事后宣判」**：提示「再这么合要死局了」（正向,帮规划）vs 告诉「你已经死了」（负向,劝退）。同一算法能力,方向不同,体验天差地别。

## 六、为什么抽卡靠「高分解锁」

**决策**：角色靠抽卡获得,但抽卡机会要靠「把关卡分玩得很高」才能解锁。

**依据**：
- 随机 ≠ 廉价成长。抽卡的随机性创始人不排斥;但「花钱/等时间就能抽」是廉价的。
- 「玩得好才有资格抽」→ 抽卡机会是实力换来的,不是白给 → 符合「不信廉价成长」。
- 形成实力驱动的正循环：玩得好 → 抽得多 → 角色全 → 更能玩高分。

**陷阱**：见第五条——角色能力不能是门槛,否则「没抽到=玩不转」会伤害地板。

## 七、为什么主题用水果（消除元素）+ 角色独立做二次元人物

**决策**:消除元素用**水果**(单维:相同水果消除);**角色与主题解耦**,角色是**独立的二次元人物**(承载技能/铭文/收集/抽卡),不是拟人水果。风格偏二次元。

**为什么水果做消除元素**:
- **零教育成本**:全人类认识水果,比抽象符号更直觉,上手零门槛(消除要的就是极低门槛)。
- **零赌博联想 + 投放友好**:分级最低,买量素材无限制,不影响投放(与动物同级的干净)。
- **零侵权**:水果是公共题材,无专属 IP,不撞大厂强 IP(开心消消乐=动物、Candy Crush=糖果)。
- **美术省成本**:水果形象现成、简单、易萌系化,外包便宜。
- **代价(已正视)**:① 颜色易撞(草莓/樱桃/苹果偏红)→ 靠形状轮廓区分,小尺寸高辨识是美术第一优先级(见 `06`)② 市场撞得多(水果消除/合成满地都是)→ **差异化不靠美术,靠算法生成的"活关卡"+ 个性化 + 可竞技**(护城河),玩家记住的是"这游戏怎么这么懂我",不是"水果"。

**为什么角色独立做二次元人物(不是拟人水果)**:
- **水果难角色化**:一个水果很难有"性格"去挂技能("香蕉=借贷"很牵强)。而你的 Meta 重度依赖角色化(借贷/回退/重力翻转等技能要挂在有性格的载体上)。
- **解耦解决错配**:消除元素=水果(享受零教育/省成本),角色=独立二次元人物(承载技能/收集/抽卡,角色化天然成立)。这是品类标准做法(Candy Crush 的糖果与 Tiffi 是两层)。
- **二次元 + 重 Meta 天然契合**:二次元用户正吃"角色收集 + 抽卡 + 养成"——你的重 Meta(4流派角色 + 铭文挂角色 + 收集)在二次元风格下不是负担,是卖点。
- **代价**:要做两套美术(水果元素 + 二次元角色立绘),且二次元立绘审美门槛高、成本高(需预算)。换来角色深度 + 收集 + 品牌,划算。

**陷阱**:
- 水果种类数同屏别太多(4-6 种,守 low floor);颜色易撞,务必靠形状区分。
- 水果与角色**风格要统一**(水果萌系 + 角色二次元立绘,别割裂)。
- 角色立绘是成本大头和审美门槛,这是收集/抽卡的卖点,不能省。
- 皮肤(尤其角色皮肤)纯外观,绝不带数值(见第八条)。

## 八、为什么变现是「广告 + 抽卡 + 皮肤」

**决策**：三条腿——广告（大众）、抽卡角色（玩法能力,靠高分解锁）、皮肤（纯外观）。

**依据**：
- 全部「不卖强度、不破坏公平、不违背不信廉价成长」。
- 皮肤是最干净的变现：只卖外观,零影响平衡,喂「to be somebody」（外显炫耀,高手顶稀有皮上榜）,且无限可做、零平衡负担。

**陷阱**：
- **皮肤必须纯外观,绝不带任何数值/能力加成。** 一旦沾强度,就从「卖审美」滑向「卖强度」,破坏公平、违背原则。
- 皮肤管外观,抽卡角色管能力,两者严格分开,别混。

## 九、贯穿一切的：算法是护城河,但藏在底层

**决策**：用算法做「关卡生成 + 难度量化 + 个性化难度」,但玩家感觉不到算法。

**依据**：
- 这是创始人团队的真实优势（算法/竞赛背景）,也是别人抄不走的壁垒。
- 同类游戏的关卡多靠人工手搓、难度靠粗放 A/B;算法能做到无限生成 + 千人千面的难度匹配,这是质的差异。

**怎么用**：
- 验证关卡有解（保 low floor,不死局）。
- 算出关卡最优解（定义 high ceiling）。
- 量化难度（用于个性化推送、分级）。
- 个性化：给累的人轻松局,给高手有挑战的局。

**陷阱**：
- 别把算法的「优化/最优解」摆到大众面前当门槛——那会劝退。算法藏在底层,大众只体验到「这游戏怎么这么舒服/这么懂我」。
- 这正是 Block Blast 没重点做的方向（它 high ceiling 靠玩法本身、meta 较轻、变现靠广告）——用算法做更深的 meta 和更精的关卡,是本项目相对它的差异化空间。

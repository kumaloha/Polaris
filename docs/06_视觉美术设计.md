# 06 视觉 / 美术设计文档

> 给美术团队。定义主题、风格、魔法道具元素、二次元角色、皮肤的视觉规格。
> **主题与角色解耦**:消除元素=魔法道具(公共题材、零教育、零侵权);角色=独立的二次元人物(承载技能/收集/抽卡)。
> 风格:**偏二次元**(萌系)。设计原则对照 `01`,Meta 见 `02`,玩法见 `04`。

---

## 一、美术总目标

**零教育成本 + 小尺寸高辨识 + 二次元审美 + 角色可收集。**
- **消除元素(魔法道具)**:零教育、一眼分种类、小图标高辨识。
- **角色(二次元人物)**:有辨识度、能暗示技能、立绘达到二次元用户审美门槛(收集/抽卡的卖点)。
- **风格统一**:魔法道具做成二次元萌系,与角色立绘风格协调,不割裂。

---

## 二、整体风格:二次元萌系

- 明亮、友好、精致的二次元/动漫画风。
- **两层资产风格要统一**:魔法道具(萌系拟物/轻拟人)+ 角色(二次元人物立绘),色调、线条、质感一致,避免"精致角色 + 廉价道具"的割裂。
- **二次元用户挑画风、挑立绘质量** → 角色立绘是审美门槛所在,需重点投入(美术成本的大头)。
- 买量友好:二次元角色 + 萌系魔法道具,在 3 秒素材里吸睛。
- 避免:写实(辨识难、不萌)、暗黑(不适配休闲大众)、过度细节(小尺寸看不清)。

> 二次元风格与本项目重 Meta(角色收集+抽卡+养成)天然契合——二次元用户正吃这套。代价是角色美术要求/成本高,需预算。

---

## 三、消除元素:魔法道具（类别维度，单维匹配）

### 3.1 匹配维度
- **消除按「魔法道具种类」单维匹配**(3 个同种道具横竖相连即消)。经典消除体验,门槛最低(见 `04`)。

### 3.2 元素库(20 种，按"轮廓形状"分组，确保小图标高辨识)
> 选这 20 个的核心逻辑:**轮廓两两不同**,小图标+快速游戏下靠形状就能分,不靠颜色(上一版水果栽在撞色)。

- **细长型**:① 药水瓶(圆肚细颈) ② 蜡烛(细柱+火苗) ③ 法杖(细长+顶星) ④ 羽毛笔(细长带弧)
- **方/块型**:⑤ 魔法书(厚方块带书脊) ⑥ 符文方碑(正方刻符) ⑦ 符咒卡(扁长方纸片)
- **圆/球型**(全场同屏最多放 1 个圆球,避免撞):⑧ 水晶球(透明球+座) ⑨ 魔力珠(纯圆宝珠) ⑩ 魔铃(圆钟带口)
- **尖角/星型**:⑪ 星之碎片(五角星) ⑫ 宝石(菱形切割) ⑬ 冰晶(六角雪花)
- **不规则/自然型**:⑭ 魔菇(伞状) ⑮ 符骨(不规则骨片) ⑯ 符文石(圆润石块刻纹) ⑰ 草药束(枝叶束)
- **容器/特殊型**:⑱ 炼金烧瓶(三角锥瓶,区别于药水瓶) ⑲ 魔法灯(提灯) ⑳ 符珠串(线性串珠)

### 3.3 同屏规则(根除撞色/撞形)
- **单关同屏 5-7 种**(消除游戏上限,再多凑不出消除也分不清),从不同形状组各抽。
- **绝不让两个"圆球型"(水晶球/魔力珠/魔铃)同屏超过一个**;形状最接近的(药水瓶 vs 烧瓶、符文方碑 vs 魔法书)不同屏。
- **颜色只作辅助**(紫/蓝/红/绿/金/青),主区分永远靠形状。
- 测试标准:同屏 5-7 种的小缩略图给没玩过的人看,应能瞬间区分每一种。

### 3.4 特效道具的视觉（接多连特效，见 04）
- **直线特效**(4连):该道具带"光束/箭头"指示消整行/列。
- **爆炸特效**(T/L):该道具带"魔法光爆/裂纹"指示范围爆炸。
- **彩球**(5连):一个醒目的"虹光魔法阵球",全彩流光,明显区别于所有普通道具。
- 特效体要**一眼区别于普通道具**(玩家立刻认出"这是个特效")。

## 四、障碍的视觉（少数几种，二次元萌系包装，见 04）

吸收的少数障碍(冰块/锁/果酱/底层目标),用萌系视觉包装,但**保持"障碍"可读性**:
- **冰块/糖霜**:道具被"冰/糖壳"包住,层数用视觉厚度/裂纹表示(N 层一眼可辨还剩几层)。
- **锁**:道具被"锁链/笼"锁住。
- **果酱**:道具被半透明"果酱"裹住。
- **底层目标(果冻类)**:格子底色,消除覆盖时一眼看到"清掉了一层"。
> 障碍视觉要清晰传达"它挡路、要清掉、还剩几层",别为了萌牺牲功能可读性。

---

## 五、角色设计（独立二次元人物，承载技能）★

### 5.1 定位
- 角色 = **独立的二次元人物**(独立人物,不是拟人道具),承载技能 + 铭文养成 + 收集 + 抽卡。
- **非叙事载体**(不做剧情);靠立绘 + 一句话能力标签传达,不靠故事。

### 5.2 视觉要求
- **二次元立绘**:精致、有辨识度、符合二次元审美(收集/抽卡卖点,投入重点)。
- **外观暗示能力**:让玩家能联想到技能(如"借贷"角色带商人/契约气质,"重力翻转"角色带颠倒/魔法气质,"时间回退"角色带时间/沙漏元素)。能力靠视觉直觉 + 图标 + 短标签,零叙事。
- **稀有度梯度**(接抽卡):普通/稀有/高稀有角色,视觉华丽度递增,服务收集与抽卡。

### 5.3 首发角色(对应 4 流派 + 强度分层，见 02)
- 借贷(核心,首抽必得)、时间回退、存档快照、隔位对换、彩球护盾、重力翻转。
- 每个角色:二次元立绘 + 技能图标 + 一句话标签。
- 后续 LiveOps 持续出新角色(收集广度/新鲜感的主力来源)。

---

## 六、皮肤设计（变现，纯外观）

### 6.1 铁律
- **纯外观,绝不带任何数值/能力加成**(见 `01`/`02`)。

### 6.2 皮肤对象
- **角色皮肤为主**(二次元角色换装,二次元用户最买单)——节日/主题/联动皮。
- (魔法道具元素皮肤为辅,可选:换一套魔法道具美术风格。)
- **换皮仍可识别**:角色换装后仍认得是哪个角色;道具换皮后仍认得是哪种道具。

### 6.3 作用
- 喂 to be somebody(角色稀有皮 + 高分上榜);长期变现 + 新鲜感。

## 七、UI 视觉调性

- 二次元萌系,与角色/魔法道具统一。
- 信息清晰:分数、目标、步数、所带角色头像 + 技能按钮,清楚但不喧宾夺主(核心是道具盘面)。
- 界面布局见 `07_UIUX设计.md`。

## 九、给美术的关键提醒

1. **小尺寸高辨识是第一优先级**(尤其魔法道具)——颜色易撞,靠形状轮廓区分;同屏 4-6 种刻意拉开形状(第三节)。
2. **两层风格统一**:道具萌系 + 角色二次元立绘,色调线条质感一致,别割裂(第二节)。
3. **角色立绘是审美门槛和成本大头**——二次元用户挑画质,这是收集/抽卡卖点,重点投入(第二/五节)。
4. **角色外观要暗示能力**(第五节)——零叙事传达技能。
5. **皮肤纯外观、换皮仍可识别**(第六节)。
6. **障碍视觉别为萌牺牲功能可读性**(第四节)。

## 十、界面清单（全游戏 6 个界面）+ 提示词

> 美术/原型按这 6 个界面产出;每个界面的详细生成提示词见 `claude_design_提示词_全界面.md`(可直接喂 Claude Design)。
> 统一基调:二次元萌系魔法风、浅紫/星空蓝、圆角卡片、明亮干净。★全程禁止卖强度道具★。

| # | 界面 | 关键内容 | 对应文档 |
|---|---|---|---|
| ① | **对局**(关卡内) | 异形棋盘 + 同屏 5-7 种魔法道具 + 特效 + 障碍 + 步数 + 多目标 + 借贷技能 | 04/07/09 |
| ② | **首页/主菜单** | 角色立绘门面 + 开始游戏 + 角色/商店/排行榜入口 + 资源栏 | 02 |
| ③ | **地图选关** | 蜿蜒路径地图 + 关卡节点三态(通关/当前/锁) + 难度节奏(周期高潮关) | 08 |
| ④ | **抽卡** | 华丽召唤魔法阵 + 单抽/十连 + 魔法水晶 + 首抽必得借贷 | 02 |
| ⑤ | **角色详情**(含铭文)★ | 大立绘 + 技能(借贷/还债/一局一次) + **4条铭文线配置(挂该角色、有限点数取舍)** | 02 |
| ⑥ | **结算** | 星级 + 得分 + 铭文碎片/水晶(高分多得) + 下一关 + 可选看广告翻倍 | 02 |

**核心循环(界面串联)**:
首页 → 开始 → 地图选关 → 对局 → 结算(得碎片+水晶) → 抽卡(抽角色)/角色详情(养铭文) → 带更强角色再玩 → 上排行榜。

**异形棋盘(对局界面要点)**:棋盘非固定矩形,每关形状不同(不规则边界/挖洞/分区),由算法生成(呼应 `08` 战场切割/`09` 生成),体现关卡多样性。

**界面级硬约束(贯穿全部)**:
- 无任何"道具栏/booster"(锤子/+步数/定位),不可购买不可囤积。
- 商店只卖皮肤(纯外观)+ 看广告入口;抽卡只出角色;结算奖励靠高分多得。
- 唯一"额外能力"=角色技能(一局一次,第二次看广告)。

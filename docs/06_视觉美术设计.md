# 06 视觉 / 美术设计文档

> 给美术团队。定义主题、风格、水果元素、二次元角色、皮肤的视觉规格。
> **主题与角色解耦**:消除元素=水果(公共题材、零教育、零侵权);角色=独立的二次元人物(承载技能/收集/抽卡)。
> 风格:**偏二次元**(萌系)。设计原则对照 `01`,Meta 见 `02`,玩法见 `04`。

---

## 一、美术总目标

**零教育成本 + 小尺寸高辨识 + 二次元审美 + 角色可收集。**
- **消除元素(水果)**:零教育、一眼分种类、小图标高辨识。
- **角色(二次元人物)**:有辨识度、能暗示技能、立绘达到二次元用户审美门槛(收集/抽卡的卖点)。
- **风格统一**:水果做成二次元萌系,与角色立绘风格协调,不割裂。

---

## 二、整体风格:二次元萌系

- 明亮、友好、精致的二次元/动漫画风。
- **两层资产风格要统一**:水果(萌系拟物/轻拟人)+ 角色(二次元人物立绘),色调、线条、质感一致,避免"精致角色 + 廉价水果"的割裂。
- **二次元用户挑画风、挑立绘质量** → 角色立绘是审美门槛所在,需重点投入(美术成本的大头)。
- 买量友好:二次元角色 + 萌系水果,在 3 秒素材里吸睛。
- 避免:写实(辨识难、不萌)、暗黑(不适配休闲大众)、过度细节(小尺寸看不清)。

> 二次元风格与本项目重 Meta(角色收集+抽卡+养成)天然契合——二次元用户正吃这套。代价是角色美术要求/成本高,需预算。

---

## 三、消除元素:水果（类别维度，单维匹配）

### 3.1 匹配维度
- **消除按「水果种类」单维匹配**(3 个同种水果横竖相连即消)。经典消除体验,门槛最低(见 `04`)。

### 3.2 数量
- **素材库总量:约 20 种水果**(常见、辨识度高的:苹果/橙子/香蕉/葡萄/西瓜/草莓/樱桃/桃子/梨/菠萝/芒果/柠檬/猕猴桃/椰子/火龙果/牛油果/蓝莓/石榴/荔枝/杨桃 等)。
- **单关同屏:4-6 种**(从库里抽)。不同关卡用不同水果组合 → 多样性 + 算法生成关卡的素材。
> 注:水果数量有限(约 20),够用;长期 LiveOps 的"新鲜感"主要靠**新角色**(角色可无限扩),水果元素不必无限扩。

### 3.3 关键:小尺寸高辨识（水果的坑，必须解决）
- **水果颜色易撞**(草莓/樱桃/苹果偏红,青提/柠檬/牛油果偏绿)。**手机小图标上,主要靠「形状轮廓」区分,不能只靠颜色。**
- 选同屏 4-6 种时,**刻意拉开形状差异**(如西瓜=大圆条纹 / 香蕉=长弯 / 葡萄=串 / 草莓=带籽心形),避免一屏出现多个"红色小圆"。
- 测试标准:把同屏水果的小缩略图给没玩过的人看,应能瞬间区分每一种。
- 萌系化(加表情/拟人)可进一步增加辨识(不同水果不同表情),但保持种类一眼可辨。

### 3.4 特效水果的视觉（接多连特效，见 04）
- **直线特效**(4连):该水果带"条纹/箭头"指示消整行/列。
- **爆炸特效**(T/L):该水果带"光爆/裂纹"指示范围爆炸。
- **彩球**(5连):一个醒目的"彩虹/全色"特殊体,与普通水果明显不同。
- 这些特效体的视觉要**一眼区别于普通水果**(玩家要立刻认出"这是个特效")。

---

## 四、障碍的视觉（少数几种，二次元萌系包装，见 04）

吸收的少数障碍(冰块/锁/果酱/底层目标),用萌系视觉包装,但**保持"障碍"可读性**:
- **冰块/糖霜**:水果被"冰/糖壳"包住,层数用视觉厚度/裂纹表示(N 层一眼可辨还剩几层)。
- **锁**:水果被"锁链/笼"锁住。
- **果酱**:水果被半透明"果酱"裹住。
- **底层目标(果冻类)**:格子底色,消除覆盖时一眼看到"清掉了一层"。
> 障碍视觉要清晰传达"它挡路、要清掉、还剩几层",别为了萌牺牲功能可读性。

---

## 五、角色设计（独立二次元人物，承载技能）★

### 5.1 定位
- 角色 = **独立的二次元人物**(不是拟人水果),承载技能 + 铭文养成 + 收集 + 抽卡。
- **非叙事载体**(不做剧情);靠立绘 + 一句话能力标签传达,不靠故事。

### 5.2 视觉要求
- **二次元立绘**:精致、有辨识度、符合二次元审美(收集/抽卡卖点,投入重点)。
- **外观暗示能力**:让玩家能联想到技能(如"借贷"角色带商人/契约气质,"重力翻转"角色带颠倒/魔法气质,"时间回退"角色带时间/沙漏元素)。能力靠视觉直觉 + 图标 + 短标签,零叙事。
- **稀有度梯度**(接抽卡):普通/稀有/高稀有角色,视觉华丽度递增,服务收集与抽卡。

### 5.3 首发角色(对应 4 流派 + 强度分层，见 02)
- 借贷(核心,首抽必得)、时间回退、存档快照、隔位对换、彩球护盾、重力翻转。
- 每个角色:二次元立绘 + 技能图标 + 一句话标签。
- 后续 LiveOps 持续出新角色(收集广度/新鲜感的主力来源)。

---

## 六、皮肤设计（变现，纯外观）

### 6.1 铁律
- **纯外观,绝不带任何数值/能力加成**(见 `01`/`02`)。

### 6.2 皮肤对象
- **角色皮肤为主**(二次元角色换装,二次元用户最买单)——节日/主题/联动皮。
- (水果元素皮肤为辅,可选:换一套水果美术风格。)
- **换皮仍可识别**:角色换装后仍认得是哪个角色;水果换皮后仍认得是哪种水果。

### 6.3 作用
- 喂 to be somebody(角色稀有皮 + 高分上榜);长期变现 + 新鲜感。

## 七、UI 视觉调性

- 二次元萌系,与角色/水果统一。
- 信息清晰:分数、目标、步数、所带角色头像 + 技能按钮,清楚但不喧宾夺主(核心是水果盘面)。
- 界面布局见 `07_UIUX设计.md`。

## 八、美术资产清单

**首发最小集**:
- [ ] 核心水果约 12-16 种(萌系)+ 各自的直线/爆炸特效形态 + 彩球
- [ ] 障碍 3 种左右(冰块/锁/底层目标)的视觉
- [ ] 首发角色 6 个(二次元立绘 + 技能图标)
- [ ] 基础 UI 套件(二次元风)
- [ ] 1-2 套角色皮肤(验证皮肤系统)

**扩充(持续)**:
- [ ] 水果扩到约 20 种
- [ ] 持续出新角色(LiveOps 主力)、新皮肤、节日/联动

## 九、给美术的关键提醒

1. **小尺寸高辨识是第一优先级**(尤其水果)——颜色易撞,靠形状轮廓区分;同屏 4-6 种刻意拉开形状(第三节)。
2. **两层风格统一**:水果萌系 + 角色二次元立绘,色调线条质感一致,别割裂(第二节)。
3. **角色立绘是审美门槛和成本大头**——二次元用户挑画质,这是收集/抽卡卖点,重点投入(第二/五节)。
4. **角色外观要暗示能力**(第五节)——零叙事传达技能。
5. **皮肤纯外观、换皮仍可识别**(第六节)。
6. **障碍视觉别为萌牺牲功能可读性**(第四节)。

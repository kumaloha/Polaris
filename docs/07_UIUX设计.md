# 07 UI / UX 交互设计文档（消除主线）

> 给交互设计。玩法见 `04`,Meta 见 `02`,美术见 `06`,界面清单+生成提示词见 `claude_design_提示词_全界面.md`。
> 前置依据:滑动/交换消除、魔法道具、多连特效、少数障碍、Meta(铭文挂角色/角色/抽卡/皮肤/排行榜)、二次元魔法风。

---

## 一、UX 总原则

- **首屏即玩**:进游戏几秒内就能消除,不被教程/菜单挡住(零教育成本)。
- **核心操作零学习**:滑动交换是全人类都会的手势。
- **即时正反馈**:消除、连锁、特效都要有爽快的视觉+音效+轻震动。
- **信息清晰不喧宾夺主**:核心是盘面,UI 元素清楚但不抢戏。

---

## 二、核心操作:滑动 / 交换

- **唯一核心操作 = 滑动交换**:手指按住一个魔法道具,滑向相邻道具 → 两者交换;若交换后形成 ≥3 连则消除,否则弹回(经典消除手感)。
- 要求:交换手感跟手、消除反馈即时。
- **消除反馈**:3 连消失 + 上方道具下落 + 顶部随机补充 + 得分跳动;连锁(补充又形成消除)要有连续的爽感递进。
- **特效反馈**:4连出直线特效、T/L出爆炸、5连出彩球——生成时有醒目动效,触发时有强反馈(尤其彩球全屏消除要够爽)。

## 四、对局界面（P0，最重要）

布局(详细生成规格见提示词合集 界面①):
- **中心:消除盘面**(占屏幕主体)——异形棋盘,魔法道具在格子里,玩家滑动交换。
- **顶部**:关卡+星级、步数、多目标栏(图标+当前/目标)、分数进度条+里程碑星。
- **底部左**:角色技能卡(借贷,体现"借了要还",一局一次,第二次看广告)。
- **底部右**:★绝不放道具栏★(无锤子/+步数/定位)——可放玩家头像/等级或留空(见硬约束)。

注意力层级:盘面(玩家 90% 注意力)> 步数/目标 > 角色技能。

### 4.3 无合法消除 → 自动洗牌（reshuffle，行业标准）
- **当盘面没有任何合法消除时(玩家卡死),自动洗牌**:重排现有道具(不增减),让玩家能继续。消除品类标准做法。
- **UI 行为**:播放轻量"洗牌"动画 + 短提示(如「重新整理棋盘」),不突兀。
- **洗牌带约束**:洗出的新布局**保证至少一个合法消除**(不是纯随机碰运气,见 `09`),洗一次必能继续。
- **不消耗步数 / 不算输**:系统兜底;只有"步数耗尽且未达标"才算输。语气中性。

---

## 五、核心循环的 UX 串联

```
首页(看到角色立绘,想要)
  → 开始 → 地图选关 → 对局(滑动消除,爽)
  → 结算(得铭文碎片+魔法水晶,"打得好得更多")
  → 抽卡(花水晶抽角色,首抽借贷) / 角色详情(养该角色铭文)
  → 带更强角色和铭文,再玩更高分 → 上排行榜被看见
```
每一步都给即时正反馈 + 明确的下一步指引。

---

## 六、渐进展开（不让新手被系统淹没）

- **第一关**:只有消除,纯玩,滑动。让他先爱上核心玩法。
- **逐步解锁**:玩几关后再引入角色技能 → 抽卡 → 铭文 → 排行榜,一次只介绍一个,别一上来堆所有系统。
- 这守住"零教育成本"和 low floor。

---

## 七、收集 / 皮肤 / 排行榜的 UX 点

- **角色收集可视化**:集齐进度(已拥有 X 个角色)、稀有度展示。
- **皮肤预览**:清晰展示"换皮肤后仍认得是哪个角色"——这是皮肤系统的信任基础。
- **排行榜**:展示高手成绩 + 稀有皮肤(服务 to be somebody);同题竞技入口。

---

## 八、操作细节

- **道具单位足够大,好滑好交换**(异形棋盘+同屏5-7种,小尺寸下也要好操作、好辨识)。
- 误操作宽容:交换不成立时平滑弹回,不惩罚。
- 触觉反馈:消除/连锁/特效配轻震动,增强爽感。

---

## 九、给交互/UI 团队的关键提醒

1. **首屏即玩,渐进展开**——别让新玩家面对一堆系统,先让他滑动消除爱上核心(一、六节)。
2. **滑动手感和消除反馈是命根子**——跟手、即时、爽(二节)。
3. **无合法消除自动洗牌(带约束),不扣步不算输**——系统兜底,语气中性(4.3)。
4. **★对局界面绝不放道具栏★**(无锤子/+步数/定位)——唯一额外能力是角色技能(四节 + 硬约束)。
5. **借贷技能 UI 要体现"借了要还"**——不是免费给好处(四节,详见提示词合集 界面⑤)。
6. **皮肤预览让玩家确认"换皮仍认得角色"**——皮肤系统的信任基础(七节)。
7. **核心是盘面**,其余 UI 清楚但不抢戏(四节)。

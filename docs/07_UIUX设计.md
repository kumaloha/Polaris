# 07 UI / UX 交互设计文档（消除主线）

> 给交互设计师。定义界面、操作流程、信息层级。
> 前置依据（**消除主线**,见 `04`/`02`/`06`）：**交换相邻两格的 3 连消除**、4连/T-L/5连特效、受限障碍、步数上限、Meta（铭文挂角色 / 角色一局一个 / 抽卡 / 皮肤 / 排行榜）。
> 设计原则对照 `01_设计哲学.md`。

---

## 一、UX 总目标

**低门槛上手 + 操作只靠手指滑动交换 + 核心循环顺畅。** 玩家打开即懂、滑动即玩;Meta 的深度藏在「玩了之后才逐步展开」,不在第一分钟堆给玩家。

核心原则：
- **首屏即玩**：进游戏几秒内就在消除,不被教程/菜单挡住。
- **渐进展开**：铭文、角色、抽卡这些 Meta 系统,随玩家进度逐步解锁/暴露,不一次性砸过来（守 low floor 的认知负担）。
- **算法藏底层**：玩家感觉「这游戏怎么这么顺/这么懂我」,但看不到任何「难度/最优解」的复杂界面。

---

## 二、核心操作：滑动交换

- **唯一核心操作 = 交换相邻两格**：手指按住一个水果,**滑向相邻格** → 两格交换。
- 规则：交换后**能形成 ≥3 连即生效**（否则弹回；含特效/彩球的交换除外,见 `04`）。
- 要求：滑动手感跟手、消除反馈即时（视觉+音效+轻震动）。
- 无需复杂手势、无需多指、无需精确点击小目标。
- **消除反馈（爽点所在）**：3 连"啵"地消失 + 下落 + 随机补充；**连锁**连续消除时逐级加分跳动 + 渐强音效；**凑出特效**（4连直线 / T-L 爆炸 / 5连彩球）有醒目生成动效,触发时大范围炸裂强反馈（high ceiling 的视觉奖励）。

> 注：本作核心操作是**交换消除**,不是"拖到另一个上合成"（那是合成 Core,见 `03`）。

## 四、消除主玩法界面（P0,最重要）

### 4.1 布局
- **中心：消除棋盘**（占屏幕主体）——水果在这里,玩家滑动交换。
- **顶部：核心状态条**——当前分数、**通关目标 + 进度**（如「冰块 8/12」）、**剩余步数**（步数告急时轻量高亮）、当前携带的角色头像。
- **角色技能按钮**（主动技能,如借贷 / 时间回退 / 重力翻转）——放在顺手位置,清晰图标 + 剩余次数（第一次免费,第二次起标「看广告续用」,见 `02`）。
- （消除**没有"待出区 / 下一个序列"**——补充随机、不预告,见 `09`。）

### 4.2 信息层级（什么显眼什么收着）
- **最显眼**：水果棋盘本身（玩家 90% 注意力在这）。
- **次级**：分数、目标进度、剩余步数。
- **收着**：铭文/角色/设置等入口（按钮小而清晰,不干扰盘面）。
- **绝不**：把难度/最优解/算法相关信息显示给玩家（藏底层）。

### 4.3 无合法消除 → 自动洗牌（reshuffle，行业标准）
- **当盘面没有任何合法消除时(玩家卡死),自动洗牌**:重排现有棋子(不增减),让玩家能继续。这是消除品类标准做法(开心消消乐/Candy Crush 都有)。
- **UI 行为**:播放一个轻量的"洗牌"动画 + 短提示(如「重新整理棋盘」),让玩家知道发生了什么,不突兀。
- **洗牌带约束**:洗出的新布局**保证至少有一个合法消除**(不是纯随机洗完碰运气,见 `09`),所以洗一次必定能继续。
- **不消耗步数 / 不算输**:洗牌是系统兜底,不扣玩家步数;只有"步数耗尽且未达标"才算输(见 `09` 的 check_lose)。
- 语气:中性、帮助,不是惩罚。

## 五、核心循环的 UX 流程（最关键的体验，对齐 `02` / `00` §6）

这是整个游戏的命脉,流程要顺畅、动力要清晰：

```
玩一局消除（滑动交换,凑连锁/特效,爽）
   ↓
局后结算：显示分数 + 铭文碎片 + 「距离下次抽卡机会还差 X 分」
   ↓
分数够高 → 解锁一次抽卡机会 + 给铭文碎片（明确的奖励感）
   ↓
抽卡：抽到角色（新能力!首抽必出「借贷」）
   ↓
带上新角色 + 给它配铭文 → 下一局能玩出新策略 / 更高分
   ↓
（攒碎片 → 强化所带角色的铭文,局面更从容）
   ↓
玩出高分 → 上周赛/排行榜,被看见
   ↓
循环
```

UX 要点：
- **结算页要清晰展示「再多 X 分就能抽卡」+「这局得了多少铭文碎片」**——把「玩好」和「奖励」的关系直接挂给玩家看,制造动力。
- **抽到角色的时刻要有仪式感**（核心奖励节点；首抽借贷让新手感到"拿到主力了"）。
- **带新角色玩第一局时,引导玩家用上技能**（让他感受到「这角色让我玩出了不一样的东西」）。

---

## 六、渐进式解锁（守 low floor）

1. **第一局**：只有消除,纯玩,滑动交换。（让他先爱上核心玩法）
2. **几局后**：解锁结算的「抽卡进度 + 铭文碎片」,暴露第一个目标。
3. **首次高分**：解锁抽卡,首抽拿到「借贷」角色。
4. **再之后**：逐步解锁铭文（给角色配）、排行榜、皮肤。

> 原则：玩家在「需要它/够格用它」之前,不暴露那个系统。一上来全暴露 = 认知过载 = 吓跑大众。

## 七、各 Meta 界面的 UX 要点（对齐 `02`）

### 铭文界面（挂在角色上,不是全局）
- **明确「在给【当前角色】配铭文」**——换角色 = 换这套铭文（见 `02` 二）。
- 4 条线（步数 / 特效 / 产出 / 战场）+ **有限点数**,清晰看到「点了什么、还能点什么」;突出「点了这条就少点那条」（取舍,对应不信廉价成长）。
- 步数线尤其传达「**靠打连锁才奖励步数**,不是白送」（见 `02` 二）。

### 角色界面
- 清晰展示「**一次只能带一个**」——选中一个,其他变灰/未携带。
- 每个角色：二次元立绘 + 技能图标 + **一句话能力说明**（含「一局一次,第二次看广告续用且有上限」,见 `02`/`06`）。
- 让玩家理解「换角色 = 换流派/换玩法 + 换那套铭文」。

### 抽卡界面
- 明确「抽卡机会靠高分解锁」+ **首抽保底借贷**（新手起点可控）；抽卡动效有仪式感（核心奖励）。

### 排行榜/竞技
- **同题竞技**：清晰展示「**同一关卡（同 seed）**,你的分 vs 别人的分」（见 `08`/`09` §5.5）。
- 展示高手成绩 + 稀有皮肤/角色皮（服务 to be somebody）；完美通关/高分标记。

### 皮肤/收集
- 收集进度可视化（角色 X、皮肤 X 套、水果种类 X/20+）。
- 皮肤预览：清晰展示「换皮肤后仍认得是哪个角色 / 哪种水果」（换皮仍可识别,见 `06`）；皮肤纯外观,界面也不暗示强度。

---

## 八、移动端注意

- 单手可玩（滑动在拇指可达范围）。
- 水果单位足够大,好按好滑（小尺寸下也要好操作；颜色易撞,靠形状区分,见 `06`）。
- 碎片时间友好（单局不要太长,适合随时玩一局）。
- 加载快、低端机也能跑（休闲大众设备参差）。

---

## 九、给交互设计师的关键提醒

1. **首屏即玩,渐进展开**——别让新玩家面对一堆系统,先让他滑动消除爱上核心。
2. **核心操作是滑动交换,不是拖拽合成**（第二节）——消除的爽点在"连锁 + 特效"的即时反馈,务必做足。
3. **无合法消除时自动洗牌(带约束,保证洗出有合法消除),不算输、不扣步**（第四节）——系统兜底,语气中性。
4. **算法/难度信息绝不暴露给玩家**——玩家只体验「顺/懂我」,看不到复杂度。
5. **铭文挂角色 + 取舍感要在 UI 上体现**（第七节）——让玩家感到「在养这个角色」且点数有限要选择（对应不信廉价成长）。
6. **皮肤预览要让玩家确认「换皮仍认得角色/水果种类」**——皮肤系统的信任基础。
7. **没有"待出区/下一个序列"**——补充随机不预告（第 4.1 节,见 `09`）。
